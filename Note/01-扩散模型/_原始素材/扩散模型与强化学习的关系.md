## 1. 核心关系概述

扩散模型与强化学习解决的是同一个问题的不同层面：**在环境中做出好的决策**。

| 维度 | 强化学习 (RL) | 扩散模型 |
|------|-------------|---------|
| **核心目标** | 最大化累积奖励 | 建模数据分布并生成样本 |
| **策略表示** | 参数化策略 $\pi_\theta(a\|s)$（通常是高斯/分类分布） | 以去噪过程隐式定义 $\pi_\theta(a\|s)$ |
| **数据来源** | 在线交互 or 离线数据集 | 通常是离线演示数据 |
| **多模态能力** | 弱（高斯策略难以表达多峰分布） | 强（天然建模多模态分布） |
| **优化信号** | 奖励函数 $r(s, a)$ | 去噪损失 $\|\epsilon - \epsilon_\theta\|^2$ |
| **探索能力** | 主动探索未知区域 | 依赖训练数据的覆盖范围 |

下面对表中三个关键维度做详细展开：

### 1.1 策略表示（Policy Representation）

策略 $\pi(a|s)$ 是"给定当前状态 $s$，用什么数学形式输出动作 $a$"——它决定了模型**能表达什么样的动作分布**。

- **RL**：显式参数化，网络直接输出均值和方差，一次采样得到动作
  $$\pi(a|s) = \mathcal{N}(\mu_\theta(s),\, \sigma^2)$$
- **扩散模型**：隐式定义，通过 K 步去噪从噪声逐步生成动作
  $$a_K \sim \mathcal{N}(0, \mathbf{I}) \;\xrightarrow{\text{去噪 K 步}}\; a_0$$

**差异举例**：桌上的杯子可以从左边抓也可以从右边抓——

- RL（高斯策略）：输出左和右的均值 → 从中间去抓 → **失败**
- 扩散模型：每次去噪可能落入左或右的模态 → 两种抓法都能生成 → **每次都合理**

### 1.2 数据来源（Data Source）

训练数据从哪里来、怎么获取，直接决定了方法的适用场景和部署成本。

| 维度 | RL | 扩散模型 |
| ------ | ------ | ------ |
| **典型来源** | 在线交互（试错采集）或离线数据集 | 离线演示数据（人类遥操作录制） |
| **获取方式** | 策略执行 → 环境返回奖励 → 反复迭代 | 人类示教一批轨迹 → 一次性训练 |
| **数据量需求** | 在线 RL 通常需要百万级交互步 | 几十到几百条高质量演示即可 |
| **安全风险** | 在线探索可能执行危险动作 | 训练阶段零风险（不需要在线交互） |

**差异举例**：训练一个机器人倒水——

- RL：机器人反复尝试，前几千次可能洒水、打翻杯子，逐步学会（在线探索的代价）
- 扩散模型：人类先遥操作示范 50 次正确倒水 → 模型直接学会模仿（不需要试错）

### 1.3 多模态能力（Multi-modality）

> 此处"多模态"**不是指视觉/语言等多种输入模态**，而是指动作概率分布中有**多个峰（mode）**——同一状态下存在多种都合理但彼此不同的动作方案。

- **RL（高斯策略）**：单峰分布，多种合理动作被平均 → 输出不属于任何合理模态的"折中"动作
- **扩散模型**：天然多峰，每次采样随机落入某个合理模态，不会产生不合理的平均

**差异举例**：机器人面前是 T 形路口，左转和右转都能到达目标——

- RL：$\mu = \frac{\text{左转} + \text{右转}}{2}$ = 直行撞墙
- 扩散模型：有时生成左转轨迹，有时生成右转轨迹，每条都合理

### 1.4 三个维度的关系

```
数据来源 → 决定了你能用什么方法
  ├─ 有演示数据 → 可以用扩散模型（离线训练）
  └─ 只有环境交互 → 必须用 RL（在线学习）

策略表示 → 决定了方法的表达能力上限
  ├─ 任务动作单峰 → 高斯策略的 RL 就够
  └─ 任务动作多峰 → 需要扩散模型的多模态能力

多模态能力 → 是策略表示的直接结果
  └─ 表达能力：扩散 > GMM > 高斯
```

**一句话总结**：扩散模型擅长"生成多样且合理的行为"，RL 擅长"通过奖励信号优化行为"。两者结合 = 既有好的行为先验，又能持续优化。

## 2. 三种协作模式

### 2.1 模式一：扩散模型作为 RL 的策略表示

传统 RL 的策略通常是高斯分布：

$$\pi(a|s) = \mathcal{N}(\mu_\theta(s), \sigma^2)$$

这在动作分布是多模态时会失效。例如：桌上的杯子可以从左边抓也可以从右边抓，高斯会输出两者的均值——从中间去抓，这往往是错误动作。

**Diffusion-QL**（Wang et al., 2023）直接用扩散模型替代高斯策略：

```
传统 RL:       s → MLP → μ, σ → 采样 a ~ N(μ, σ²)
Diffusion-QL:  s → 扩散去噪过程 → 生成 a（经过 K 步去噪）
```

训练目标变为：

$$\mathcal{L} = \underbrace{\mathbb{E}\left[\|\epsilon - \epsilon_\theta(a_t, s, t)\|^2\right]}_{\text{行为克隆项（扩散损失）}} + \lambda \underbrace{Q_\phi(s, a_0)}_{\text{RL项（Q值引导）}}$$

即：既要拟合演示数据的分布，又要朝 Q 值高的方向偏移。

**代表工作**：

| 工作 | 方法 | 特点 |
|------|------|------|
| Diffusion-QL | 扩散策略 + Q值正则 | 离线 RL 中的策略约束 |
| QGPO | Q值引导的扩散策略优化 | 用 Q 函数做 classifier guidance |
| EDP | 能量引导的扩散策略 | 灵活的能量函数引导 |

### 2.2 模式二：扩散模型作为 RL 的规划器（Planner）

这是 **Diffuser**（Janner et al., 2022）的核心思路，对应传统 Model-Based RL 中"世界模型 + 规划"的范式：

```
传统 Model-Based RL:
  世界模型 p(s'|s,a) → 前向模拟 → 在模拟轨迹上做规划（CEM / MPPI / iLQR）

Diffuser 方式:
  扩散模型直接生成完整轨迹 τ = (s₀,a₀,s₁,a₁,...,sT,aT)
  通过 classifier guidance 用奖励/Q函数引导轨迹向高回报方向
```

**关键优势**：Diffuser 把世界模型和规划器统一成一个生成过程，不需要分开训练。

**工作原理**：

```
                   噪声轨迹 τ_T ~ N(0, I)
                          │
                          ▼
              ┌───────────────────────┐
              │  扩散去噪 + 奖励引导    │
              │                       │
              │  ε̃ = ε_θ(τ_t, t)     │
              │      - w·∇_τ J(τ)    │  ← J(τ) 是奖励/Q值
              │                       │
              └───────────┬───────────┘
                          │  重复 K 步
                          ▼
              去噪后的轨迹 τ_0 = (s₀,a₀,...,sT,aT)
                          │
                          ▼
                  执行 a₀，观测 s₁，重新规划
```

### 2.3 模式三：扩散模型作为离线 RL 的正则化器

离线 RL 的核心挑战是**分布偏移（Distribution Shift）**——策略不能执行数据集中未见过的动作，否则 Q 函数的估计会不准确。

扩散模型天然可以约束策略在数据分布内：

```
离线数据集 D → 训练扩散模型 p_θ(a|s) 拟合行为策略
                        ↓
推理时: 从 p_θ(a|s) 采样多个候选动作 {a¹, a², ..., aⁿ}
                        ↓
用 Q 函数 Q(s,a) 对候选动作打分，选最优的执行
```

**代表工作**：

| 工作 | 方法 | 特点 |
|------|------|------|
| SfBC | 扩散行为克隆 + Q值加权采样 | 简单高效 |
| IDQL | 扩散先验 + 隐式 Q 学习 | 避免策略外评估 |
| Decision Diffuser | 条件扩散 + 回报条件 | 用目标回报做条件引导 |

## 3. 机器人应用中的协作架构

### 3.1 层次化协作框架（最主流）

```
┌────────────────────────────────────────────────────────────┐
│  高层：任务规划层（LLM / Task Planner）                      │
│  输入: 自然语言指令 "把杯子放到盘子上"                        │
│  输出: 子目标序列 [抓取杯子] → [移动到盘子上方] → [放下杯子]   │
│  技术: LLM (GPT-4 / LLaMA) + 场景理解                      │
└────────────────────────────┬───────────────────────────────┘
                             ▼
┌────────────────────────────────────────────────────────────┐
│  中层：运动规划层（Diffusion Model）                         │
│  输入: 子目标 + 视觉观测 (RGB/点云) + 机器人状态              │
│  输出: 动作轨迹 [a_t, a_{t+1}, ..., a_{t+H}]               │
│  技术: Diffusion Policy / Diffuser                         │
│  优势: 多模态动作生成、时序一致性、平滑轨迹                    │
└────────────────────────────┬───────────────────────────────┘
                             ▼
┌────────────────────────────────────────────────────────────┐
│  底层：控制执行层（RL / 传统控制）                             │
│  输入: 目标轨迹 + 实时传感器反馈（力/力矩、关节状态）           │
│  输出: 关节力矩 / 电机指令                                   │
│  技术: RL残差策略 / 阻抗控制 / PID                           │
│  作用: 鲁棒执行、力控制、sim-to-real 适应                     │
└────────────────────────────────────────────────────────────┘
```

### 3.2 各层职责对比

| 层级 | 核心问题 | 时间尺度 | 扩散模型角色 | RL 角色 |
|------|---------|---------|------------|--------|
| 任务层 | 做什么 (What) | 秒~分钟 | - | - |
| 规划层 | 怎么做 (How) | 100ms~秒 | **主导**：生成动作轨迹 | 辅助：提供 Q 值引导 |
| 控制层 | 执行 (Execute) | 1~10ms | 辅助：提供参考轨迹 | **主导**：实时控制与适应 |

## 4. 具体协作场景

### 4.1 场景 A：Diffusion Policy + RL 微调（最常见）

```
阶段1: 人类演示 → 训练 Diffusion Policy（模仿学习/行为克隆）
       损失: L = E[||ε - ε_θ(a_t, o, t)||²]
                        ↓
阶段2: 在真实/仿真环境中用 RL 微调
       损失: L = L_diffusion + λ · L_RL
       - 奖励信号: 任务成功率 + 安全约束
       - 扩散策略提供好的初始化，RL 提升鲁棒性和成功率
```

**DPPO（Diffusion Policy Policy Optimization, 2024）** 是这一思路的代表工作：

- 先用演示数据训练 Diffusion Policy
- 再用 PPO 算法对扩散策略做在线微调
- 核心创新：将扩散模型的去噪过程展开成一个**"虚拟 MDP"**

```
虚拟 MDP 视角下的 DPPO:

真实环境状态 s → 初始噪声 a_K ~ N(0,I)
                    │
                    ▼  去噪步骤 K (视为虚拟 MDP 的第一步)
                 a_{K-1}
                    │
                    ▼  去噪步骤 K-1
                   ...
                    │
                    ▼  去噪步骤 1 (视为虚拟 MDP 的最后一步)
                   a_0 → 在真实环境中执行 → 获得奖励 r

PPO 梯度沿整个去噪链反向传播
```

### 4.2 场景 B：扩散世界模型 + RL 策略学习

```
真实少量数据 → 训练扩散世界模型 p_θ(s', r | s, a)
                        ↓
在扩散世界模型中"想象"（rollout）大量轨迹
                        ↓
用想象轨迹做 RL 训练（类似 Dreamer / MBPO）
                        ↓
策略部署到真实环境
```

**代表工作**：

- **UniSim**：用视频扩散模型做通用世界模拟器
- **Genie**：从互联网视频学习交互式世界模型
- **SuSIE**：用扩散模型做子目标图像生成，RL agent 学习到达子目标

**优势**：

- 大幅减少真实交互次数（sample efficiency）
- 扩散世界模型比传统世界模型生成的轨迹更逼真
- 可以生成多样化的场景做 domain randomization

### 4.3 场景 C：多机器人协调

在多机器人场景中，扩散模型和 RL 各自承担不同的角色：

```
┌──────────────────────────────────────────────────────┐
│          中央扩散规划器 (Centralized Planner)           │
│                                                      │
│  输入: 所有机器人的状态 + 全局任务目标                    │
│  输出: 所有机器人的联合轨迹                              │
│  方法: 类似 MotionDiffuser，建模联合分布                 │
│        p(τ¹, τ², ..., τⁿ | goal)                     │
│                                                      │
│  优势: 天然建模多机器人交互（避让、协作、同步）             │
└────────────┬─────────────┬──────────────┬────────────┘
             ▼             ▼              ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
    │ 机器人1       │ │ 机器人2       │ │ 机器人3       │
    │ 分布式 RL     │ │ 分布式 RL     │ │ 分布式 RL     │
    │ 控制器        │ │ 控制器        │ │ 控制器        │
    │              │ │              │ │              │
    │ 局部观测 +    │ │ 局部观测 +    │ │ 局部观测 +    │
    │ 目标轨迹 →   │ │ 目标轨迹 →   │ │ 目标轨迹 →   │
    │ 低层控制动作  │ │ 低层控制动作  │ │ 低层控制动作  │
    └──────────────┘ └──────────────┘ └──────────────┘
```

**分工**：

- **扩散模型**：全局协调，生成避碰且协同的联合轨迹
- **RL**：局部执行，处理实时干扰和接触力控制

### 4.4 场景 D：扩散数据增强 + RL 训练

```
少量真实演示 → 扩散模型生成大量增强数据
              （变换视角、光照、物体纹理、背景等）
                        ↓
增强后的大规模数据集 → 训练 RL 策略
                        ↓
策略获得更好的泛化能力
```

**代表工作**：GenAug、RoboGen、ROSIE

## 5. 方法选择指南

### 5.1 场景 → 方法映射

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 有大量高质量演示数据 | Diffusion Policy（纯模仿学习） | 扩散模型直接拟合演示分布，效果好 |
| 演示数据少但有仿真器 | RL + 扩散世界模型 | RL 在扩散世界模型中做大量想象训练 |
| 需要超越演示水平 | Diffusion Policy + RL 微调 | 扩散提供初始化，RL 通过奖励优化 |
| 动作分布简单（单峰） | 传统 RL 即可 | 不需要扩散模型的多模态建模能力 |
| 实时性要求极高（< 1ms） | 传统 RL / 传统控制 | 扩散去噪仍然较慢（通常 10-100ms） |
| 多智能体协调 | 扩散规划器 + 分布式 RL | 扩散建模联合分布，RL 做局部适应 |
| 离线数据集质量参差不齐 | 扩散正则化的离线 RL | 扩散约束策略在数据支撑内 |
| 长时序复杂任务 | LLM + Diffusion + RL 层次化 | 各层各司其职 |

### 5.2 决策流程图

```
开始
 │
 ├─ 有高质量演示数据？
 │   ├─ 是 → 演示是否足够覆盖所有场景？
 │   │        ├─ 是 → Diffusion Policy（纯模仿学习）
 │   │        └─ 否 → Diffusion Policy + RL 微调
 │   └─ 否 → 有仿真器？
 │            ├─ 是 → 扩散世界模型 + RL 训练
 │            └─ 否 → 传统 RL（在线学习）
 │
 ├─ 动作空间是否多模态？
 │   ├─ 是 → 必须用扩散模型（或其他多模态策略如 GMM）
 │   └─ 否 → 高斯策略的传统 RL 可能足够
 │
 └─ 实时性要求？
     ├─ < 1ms → 传统控制 / 轻量 RL
     ├─ 1-100ms → Consistency Distillation 后的扩散策略
     └─ > 100ms → 标准 Diffusion Policy
```

## 6. 前沿趋势（2024-2025）

### 6.1 直接对扩散策略做策略梯度优化

- **DPPO**（Allen et al., 2024）：PPO + Diffusion Policy
- **RFGP**：奖励引导的 flow matching 策略
- 核心思想：将去噪链视为可微分的计算图，RL 梯度端到端传播

### 6.2 Foundation Model + 扩散 + RL

```
π₀ (Physical Intelligence) 的架构:

互联网规模预训练 (VLM)
         │
         ▼
视觉-语言-动作基础模型 (Foundation Model)
         │
         ▼
扩散头 (Diffusion Action Head) ← 生成多模态动作
         │
         ▼
RL Post-Training ← 针对具体任务/机器人微调
         │
         ▼
通用机器人策略
```

### 6.3 实时扩散控制

- 通过 **Consistency Distillation** 把去噪压缩到 1-2 步
- 使实时 RL 闭环控制成为可能（< 10ms 推理）
- 代表：Consistency Policy（Prasad et al., 2024）

### 6.4 扩散世界模型 + RL 大规模训练

- 用视频扩散模型做世界模型（如 UniSim、Genie 2）
- RL agent 在"想象"的视频世界中大规模学习
- 类似 AlphaGo 在模拟器中自我博弈，但世界模型是学习到的

### 6.5 安全约束下的扩散 + RL

- 扩散模型生成候选轨迹
- RL 的价值函数 + 安全 critic 进行筛选
- 形式化安全保障（CBF / 可达性分析）约束扩散采样
