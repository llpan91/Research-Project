# UMI-FT 论文解析：21D 动作向量与 Diffusion Policy 的实际价值

> 论文：*In-the-Wild Compliant Manipulation with UMI-FT*
> arXiv: 2601.09988v1, 15 Jan 2026

---

## 一、为什么输出 21 维动作向量（含 9D 位姿），而不是 6DoF？

### 1. 21 维向量的具体构成

论文 Section IV-B (Output Decoding) 明确给出了分解：

| 分量 | 维度 | 说明 |
|------|------|------|
| Reference Pose | **9D** | 3D 平移 + 6D 旋转（旋转矩阵前两行） |
| Virtual Target Pose | **9D** | 同上格式，柔顺控制器的实际设定目标 |
| Stiffness Value | **1D** | 标量，编码机械臂刚度矩阵 |
| Gripper Width | **1D** | 期望夹爪宽度 |
| Grasp Force | **1D** | 期望夹持力 |
| **总计** | **21D** | |

---

### 2. 为什么旋转用 6D 而不是 3D（欧拉角）或 4D（四元数）？

这是遵循 Diffusion Policy [30] 的惯例，其数学根源来自 Zhou et al. 的连续旋转表示理论：

- **欧拉角 (3D)**：存在**万向节锁 (Gimbal Lock)**，且角度空间存在 ±π 跳变等不连续性，对神经网络的回归学习非常不友好
- **四元数 (4D)**：存在**对径等价问题** —— `q` 和 `-q` 表示同一旋转，导致损失函数的拓扑空间不连续，网络学习时会在两个等价点间震荡
- **旋转矩阵前两行 (6D)**：这是一种**拓扑连续的表示**。取 3×3 旋转矩阵的前两行（6 个数），推理时通过 Gram-Schmidt 正交化恢复第三行，重建完整旋转矩阵。这种表示在 SO(3) 到 ℝ⁶ 的映射上是连续的，对神经网络回归最友好

> **结论**：9D = 3D position + 6D continuous rotation，本质上就是 6DoF 位姿的一种**对学习更友好的参数化形式**，不是增加了新的信息维度。

### 3. 为什么要输出「两个」位姿（Reference Pose + Virtual Target Pose）？

| 训练了一种自适应柔顺策略（Adaptive Compliance Policy），可以同时预测目标位置、抓取力和刚度，并在标准柔顺控制器上执行
这是 UMI-FT 最核心的设计之一，来源于 ACP (Adaptive Compliance Policy) [8] 框架。两个位姿配合 stiffness 标量，共同编码了一个**时变虚拟弹簧模型**：

```
                   Stiffness (k)
Reference Pose ←────/\/\/\/────→ Virtual Target Pose
  (弹簧一端)           弹簧          (弹簧另一端 = 控制器设定目标)
```

- **Reference Pose（参考位姿）**：策略认为末端执行器「应该在的位置」，即名义轨迹点
- **Virtual Target Pose（虚拟目标位姿）**：发送给底层柔顺控制器的**实际设定目标**，它与参考位姿之间的偏移量编码了期望施加的力
- **Stiffness（刚度）**：弹簧常数 `k`，决定偏移量到力的映射关系

**核心逻辑**：

```
F = k × (Virtual Target - Reference Pose)
```

**具体例子（白板擦除任务）**：

- 策略希望擦板时施加 10N 向下力
- 不直接输出"10N"，而是输出 Reference Pose 在白板表面、Virtual Target 穿透白板表面几毫米、Stiffness 设为合适的值
- 底层柔顺控制器追踪 Virtual Target，遇到白板阻挡后自然产生弹簧力，实现柔顺接触

**为什么不直接输出力？**

因为直接预测力并执行需要精确的力控制器，且纯力控制在自由空间（未接触时）没有意义。用虚拟弹簧模型，**自由空间走位置、接触后走力**，天然统一了两种行为模式。

## 二、Diffusion Policy 在这个工作中的实际价值

### 1. 系统架构中的定位

论文 Fig. 3 展示了三层控制回路：

```
┌────────────────────────────────────────────────────────────┐
│  Learned Visuomotor Policy (Diffusion Policy)    @ 1 Hz    │  ← 最慢，做"决策"
│  输入: RGB, Depth, F/T (32帧), Proprioception              │
│  输出: 21D 动作向量                                          │
├────────────────────────────────────────────────────────────┤
│  Arm Compliance Controller (Admittance)          @ 500 Hz  │  ← 中层，做"柔顺执行"
│  Grasp Force Controller                          @ 30 Hz   │  ← 底层，做"夹持力调节"
└────────────────────────────────────────────────────────────┘
```

Diffusion Policy 是**最上层的「大脑」**，以 1Hz 的频率生成包含位姿、刚度、夹持力的完整动作指令；底层两个基于模型的控制器以高频率做实际的物理交互。

---

### 2. 为什么必须用 Diffusion Policy 而非普通回归网络？

#### (a) 动作的多模态性（Multi-modality）

在灯泡插入任务中，当针脚需要对齐插槽时，机器人可以顺时针或逆时针旋转搜索——两种方向都是有效的。普通回归网络（MSE loss）会把两个模态取平均，输出一个**介于两者之间的无效动作**（比如不旋转）。Diffusion Policy 通过去噪过程对完整的动作分布建模，可以**采样到任意一个有效模态**而不会模态坍缩。

#### (b) 时序动作块预测（Action Chunk Prediction）

Diffusion Policy 不是逐帧预测单个动作，而是一次生成**一段未来动作序列**。这对柔顺操作至关重要：

- 擦白板时需要持续稳定的接触力轨迹
- 穿刺西葫芦时需要平滑的力递增过程
- 时序一致的动作块天然避免了帧间抖动

#### (c) 灵活的多模态条件输入

Diffusion Policy 本质是条件生成模型，可以自然地接受**任意多模态输入**作为条件。论文中的观测编码流程：

```
RGB (2帧)          → ViT-B/32      ──┐
Depth              → ViT-B/32      ──┤
Left F/T (32帧)    → CausalConv    ──┤→ Self-Attention Fusion → Diffusion → 21D Action
Right F/T (32帧)   → CausalConv    ──┤
Proprioception     ─────────────────┘
```

各个模态的 token 经过 Self-Attention 融合后，作为条件送入扩散过程。这种架构天然支持 UMI-FT 新增的力/力矩模态，不需要重新设计网络结构。

---

### 3. 实验中的直接证据

论文所有 baseline 对比中，Diffusion Policy 都是**固定的策略骨干**，变量是输入模态和输出执行方式：

| 方法 | 策略骨干 | 力观测 | 柔顺控制 | 白板擦除 | 灯泡插入 |
|------|---------|--------|---------|---------|---------|
| **Ours (ACP)** | Diffusion Policy | CoinFT | 有 | **92%** | **95%** |
| DP w/ F | Diffusion Policy | CoinFT | 无 | 28% | 60% |
| DP | Diffusion Policy | 无 | 无 | 16% | 0% |
| DP w/ CM | Diffusion Policy | 接触麦克风 | 无 | 0% | 20% |

- 即使在没有柔顺控制的 baseline (DP w/ F) 中，Diffusion Policy + 力观测已经能完成部分任务（灯泡 60%、西葫芦 70%），说明 Diffusion Policy 本身确实学到了**力感知的操作策略**
- 完整系统 (ACP) 的大幅提升来自**将 Diffusion Policy 的高层决策与底层柔顺控制解耦**

---

### 4. 总结：Diffusion Policy 的不可替代价值

| 价值 | 具体体现 |
|------|---------|
| **多模态动作分布建模** | 避免接触丰富任务中的模态平均问题 |
| **时序一致的动作块** | 保证力轨迹平滑，避免抖动 |
| **灵活的多模态条件输入** | 天然融合 RGB + Depth + F/T + Proprioception |
| **高层-低层解耦** | 1Hz 做决策 + 500Hz 做柔顺执行，既有智能又有反应速度 |

> Diffusion Policy 在这个工作中不是「可以替换的模块」，而是整个系统的**决策核心**——它以低频率输出「做什么」（21D 向量），底层控制器以高频率解决「怎么做」（柔顺接触）。

## 三、底层控制器如何实现高频柔顺接触

Diffusion Policy 以 ~1Hz 输出 21D 动作向量后，**真正的物理交互由两个基于模型的高频控制器完成**。这是整个系统能安全、稳定地完成力敏感任务的关键。

### 1. 总体架构：三环并行、频率分层

```
┌─────────────────────────────────────────────────────────────────┐
│ 感知输入                                                          │
│ RGB, Depth, Left/Right CoinFT (32帧), Proprioception            │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第 1 层：Visuomotor Policy（Diffusion Policy）       @ ~1 Hz     │
│ 输出: Ref Pose, Virtual Target, Stiffness, Gripper Width, Force │
└────────┬──────────────────────────────────┬─────────────────────┘
         ▼                                  ▼
┌──────────────────────────┐  ┌────────────────────────────────┐
│ 第 2 层：Wrist Compliance │  │ 第 3 层：Grasp Force Controller │
│ Controller (手臂导纳控制)  │  │ (夹爪力控制)                     │
│ @ 500 Hz                  │  │ @ 30 Hz                         │
│ 输入: Virtual Target,     │  │ 输入: 目标抓取力 f_G*,           │
│   Stiffness, F/T Wrench  │  │   CoinFT 实测抓取力 f_G          │
│ 输出: 机械臂关节指令        │  │ 输出: 夹爪速度指令               │
└──────────────────────────┘  └────────────────────────────────┘
```

**关键设计思想**：策略做低频「决策」，控制器做高频「反应」。这种分层使得：

- 策略可以从容处理高维视觉输入（1Hz 足够）
- 物理接触的力反馈在 500Hz 内闭环，远快于物体动力学变化
- 两层完全解耦，底层控制器不需要任何学习

---

### 2. 手臂层：6D 导纳控制（Admittance Control）@ 500Hz

#### 核心原理

导纳控制将机械臂末端模拟为**虚拟弹簧-质量-阻尼器（Spring-Mass-Damper）系统**：

```
外部环境力 F_ext
       │
       ▼
  ┌─────────┐     弹簧(K)      ┌───────────┐
  │ 末端执行器 │←──/\/\/\/──────│ Virtual    │
  │ (实际位置) │←──[阻尼器]─────│ Target Pose│
  │          │     阻尼(D)      │ (设定目标)  │
  └─────────┘                  └───────────┘
```

当末端执行器接触环境产生外力时，导纳控制器让末端执行器**像弹簧一样柔顺退让**，而不是刚性地强制到达目标位置。

#### 力信号融合

两个 CoinFT 传感器的 6D 力/力矩测量需要先统一到机器人工具坐标系：

$$W_{wrist} = \text{Ad}_{S_1T}^T \cdot W_{S1} + \text{Ad}_{S_2T}^T \cdot W_{S2}$$

其中：

- $W_{S1}, W_{S2}$：左右 CoinFT 的 6D 力/力矩读数（各 $\in \mathbb{R}^6$）
- $\text{Ad}_{S_iT}^T$：传感器坐标系到工具坐标系的**伴随变换矩阵的转置**（用于力旋量的坐标变换）
- $W_{wrist}$：合并后的工具坐标系力/力矩，作为导纳控制器的反馈输入

> **为什么要合并两个传感器？** 因为外力可能作用在任一手指上（如擦白板时主要是一侧接触），合并后控制器能响应任意手指上的外力。

#### Diffusion Policy 输出如何驱动导纳控制器

策略输出的三个关键量直接控制导纳行为：

| 策略输出 | 导纳控制器中的角色 | 物理含义 |
| --------- | ------------------ | --------- |
| **Virtual Target Pose** | 控制器的**设定目标位置** | 弹簧的平衡点，末端被拉向此处 |
| **Reference Pose** | 用于计算期望力 | 与 Virtual Target 的偏差编码期望施力方向和大小 |
| **Stiffness 标量** | 重建为完整的 **6×6 刚度矩阵 K** | 弹簧软硬程度；高刚度 → 位置跟踪精确；低刚度 → 柔顺退让 |

**刚度矩阵的重建过程**（遵循 ACP [8] 方法）：

- 策略只输出 1 个标量而非完整 6×6 矩阵
- 训练前通过对示教数据中的力-运动关系进行后处理，用**时变 3D 弹簧模型**拟合得到刚度和虚拟目标的标签
- 推理时，将标量重建为完整刚度矩阵，送入导纳控制器

#### 不同场景下的刚度自适应

这是 "Adaptive" Compliance Policy 名称的由来——**策略学会了根据任务阶段动态调整刚度**：

```
灯泡插入任务的刚度变化（示意）：

接近阶段:  K = 高  → 精确位置跟踪，快速接近插座
搜索阶段:  K = 低  → 柔顺探索，bayonet 针脚碰到插槽边缘时不会卡死
插入阶段:  K = 中  → 适度力施加，压缩弹簧电极
旋转阶段:  K = 低  → 柔顺旋转，保持针脚在槽内接触
```

## 三-b、ACP 与 Diffusion Policy 的关系：继承与扩展

> 核心结论：**ACP 本质上就是一种 Diffusion Policy**，但在输出空间和执行架构上做了面向力控的关键扩展。

### 1. ACP 的策略骨架 = Diffusion Policy

ACP 论文（arXiv:2410.09309）Section III-D 明确写道：

> "We formulate the policy as a **diffusion process** for both reference action and target stiffness. The policy runs in a **receding-horizon manner** [6]."

其中 [6] 就是 Cheng Chi 的 Diffusion Policy。ACP 完整继承了 Diffusion Policy 的核心机制：

| 机制 | Diffusion Policy | ACP |
|------|-----------------|-----|
| 生成模型 | 条件去噪扩散 | 条件去噪扩散（相同） |
| 动作生成方式 | 滑动窗口动作块 | 滑动窗口动作块（相同） |
| 视觉编码 | CLIP ViT | CLIP ViT（相同） |
| 多模态融合 | Transformer Self-Attention | Transformer Self-Attention（相同） |

### 2. ACP 扩展了什么？

ACP 的创新不在策略架构本身，而在**输出空间**和**执行方式**：

#### (a) 输出空间：从「纯位姿」到「位姿 + 柔顺参数」

| 输出分量 | 标准 Diffusion Policy | ACP |
|---------|----------------------|-----|
| 参考位姿 (Reference Pose, 9D) | ✓ | ✓ |
| 虚拟目标位姿 (Virtual Target Pose, 9D) | ✗ | ✓ |
| 刚度标量 (Stiffness) | ✗ | ✓ |
| 参考力 (Reference Force) | ✗ | ✓ |

标准 Diffusion Policy 每个臂输出 9D（一个位姿），ACP 每个臂输出 19D（两个位姿 + 刚度）。UMI-FT 在此基础上又加了夹爪宽度和抓持力，最终成为 21D。

#### (b) 执行架构：从「位置控制器」到「柔顺控制器」

```
标准 Diffusion Policy 执行链：
  Diffusion Policy → 目标位姿 → 位置控制器 → 机器人
                                 （刚性跟踪）

ACP 执行链：
  Diffusion Policy → 虚拟目标 + 刚度矩阵 → 柔顺控制器 → 机器人
                                            （弹性跟踪，接触时变软）
```

这正是 ACP 论文 Fig. 2 所展示的三种方法对比：
- **(a) Visuomotor Policy**：策略 → 位姿 → 位置控制器（最简单，无力感知）
- **(b) Compliance Policy**：策略 → 位姿 + 力 → 柔顺控制器（有力输入，但刚度固定）
- **(c) Adaptive Compliance Policy**：策略 → 位姿 + 力 + **时变刚度** → 柔顺控制器（刚度随任务动态调整）

### 3. 一句话总结

```
ACP = Diffusion Policy（生成骨架）
    + 柔顺参数预测（输出扩展：虚拟目标位姿 + 刚度）
    + 底层柔顺控制器（执行层替换：导纳控制替代位置控制）
```

它不是一种全新的策略架构，而是在 Diffusion Policy 上叠加了**力-柔顺维度**，使同一套扩散生成框架能处理需要精细力控的接触丰富任务。UMI-FT 进一步在 ACP 基础上增加了指尖力感知（CoinFT）和抓持力控制，形成了完整的 21D 输出。

## 四、UMI-FT 的核心贡献

论文在 Section I 末尾明确列出了四项主要贡献：

### 1. 带有逐指力/力矩感知的手持数据采集设备

UMI-FT 在 UMI 的基础上，为**每根手指**安装了一颗 CoinFT（硬币大小的六轴力/力矩传感器），同时采集 RGB、深度、位姿和力信号。相比前代工作的关键改进：

| 对比维度 | 腕部 F/T 传感器方案 | 触觉传感器方案 | **UMI-FT（CoinFT）** |
|---------|------------------|------------|---------------------|
| 感知位置 | 腕部，远离接触点 | 指尖表面 | **指尖，贴近接触区域** |
| 外力 vs 内力 | 仅外力 | 仅外力（接触压力） | **外力 + 内部抓持力** |
| 测量类型 | 标准六轴力/力矩 | 压力分布/滑移等 | **标准六轴力/力矩（可校准到 SI 单位）** |
| 成本 | 高（数千美元） | 中等 | **低（~$10 BOM/颗）** |
| 鲁棒性 | 脆弱 | 视实现而定 | **抗冲击、可更换** |

**核心价值**：人类操作者在示教时自然施加的抓持力和接触力，终于可以被**完整、准确地记录**下来，而不仅仅是视觉和位姿信息。

### 2. 面向柔顺操作的控制架构

将 ACP（Adaptive Compliance Policy）框架与 UMI-FT 结合，构建了**三环控制回路**：

- **视觉运动策略**（1 Hz）：输出 21D 动作向量（位姿、刚度、夹持力）
- **手腕柔顺控制器**（500 Hz）：基于导纳控制，利用双指 CoinFT 的合力反馈调节外部接触力
- **夹持力控制器**（30 Hz）：利用双指 CoinFT 读数差调节内部抓持力

这套架构的独特之处在于：**不需要昂贵的腕部 F/T 传感器**，仅靠两颗指尖 CoinFT 即可同时实现外力柔顺和内力调节。

### 3. 实验验证：力感知 + 柔顺控制的显著优势

在三个接触丰富的任务上进行了系统性消融实验：

| 任务 | Ours (ACP) | DP w/ F（有力无柔顺）| DP（无力无柔顺）| DP w/ CM（接触麦克风）|
|------|-----------|-------------------|---------------|-------------------|
| 白板擦除 | **92%** | 28% | 16% | 0% |
| 穿刺西葫芦 | **80%** | 70% | 30% | — |
| 灯泡插入 | **95%** | 60% | 0% | 20% |

实验揭示的关键结论：
- **力观测本身就有价值**（DP → DP w/ F 的提升），但仅有力观测不够
- **柔顺控制不可或缺**（DP w/ F → ACP 的跨越式提升），特别是在需要精确力调节的场景
- **接触麦克风不适合持续力调节**，仅能检测接触事件而无法量化力的大小
- **野外数据带来更好的泛化**：穿刺任务中，用 15 个野外场景的 630 条数据训练达到 100% 成功率，而仅用实验室数据仅 20%

完整开源了硬件设计和软件栈（https://umi-ft.github.io/），降低了社区在多模态力感知数据采集领域的准入门槛。

## 五、UMI-FT 关键模块的输入输出与设计细节

### 1. 硬件：UMI-FT 数据采集设备

#### 设计目标

在手持设备上实现**视觉 + 深度 + 力/力矩 + 位姿**的同步多模态采集，同时保持便携性和低成本。

#### 组成与传感器规格

```
┌─────────────────────────────────────────────────┐
│                  UMI-FT 设备                      │
│                                                   │
│  ┌──────────┐    ┌───────────────────────────┐   │
│  │ iPhone   │    │  手指结构（左 / 右）        │   │
│  │ 15 Pro   │    │  ┌───────┐  ┌───────┐     │   │
│  │          │    │  │CoinFT │  │CoinFT │     │   │
│  │ 15° 倾斜 │    │  │(左指) │  │(右指) │     │   │
│  └──────────┘    │  └───────┘  └───────┘     │   │
│                   └───────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

| 传感器 | 规格 | 采样率 |
|-------|------|-------|
| 主 RGB | iPhone 主摄，~80° FoV | 60 Hz |
| 超广角 RGB | iPhone 超广角，120° FoV | 10 Hz |
| 深度 | iPhone LiDAR | 30 Hz |
| 位姿 | ARKit | 60 Hz |
| CoinFT × 2 | 六轴力/力矩，20mm∅ × 3mm，2g | 360 Hz |

**关键硬件改动**（相对于原始 UMI）：
- **鳍状指结构重设计**：CoinFT 在压缩载荷下鲁棒，但拉伸易分层。将传感器安装位置移至指尖靠近端，使夹持主要产生压缩力而非拉伸力
- **CoinFT 介电层改型**：椭圆截面柱阵列（长轴 4mm，短轴 0.5mm）替代原有圆柱（∅0.2mm），牺牲部分灵敏度换取更大量程和鲁棒性

#### 数据同步

CoinFT 通过 I²C 连接 USB-串口桥，用互联网时间戳与 iPhone 数据流在后处理中对齐。

### 2. CoinFT 手指标定模块

#### 输入 → 输出

```
原始电容值（CoinFT 输出）  ──→  [MLP 标定模型]  ──→  六轴力/力矩（Fx, Fy, Fz, Tx, Ty, Tz）
```

#### 为什么需要标定

CoinFT 基于电容感知原理，其信号与力的关系是**非线性的**，且受手指结构的机械耦合影响，无法用简单线性模型建模。

#### 标定流程

1. **原位安装**：将 CoinFT 安装到 UMI-FT 手指上再做标定，确保标定模型捕获完整的机械结构效应
2. **真值采集**：使用商用 Gamma（ATI）F/T 传感器作为参考标准
3. **随机加载**：人工施加随机组合的力和力矩，覆盖法向 25N、剪切 20N、力矩 500mNm 范围
4. **MLP 训练**：5 层全连接网络（128-64-36-24-12），ReLU 激活，输入电容值，输出 6D 力/力矩

#### 标定精度

| 分量 | MSE |
|------|-----|
| Fx, Fy | 0.18 N, 0.15 N |
| Fz | 0.58 N |
| Tx, Ty | 159 mNm, 231 mNm |
| Tz | 17 mNm |

### 3. 观测编码模块（Observation Encoding）

#### 各模态的编码路径

```
输入模态               编码器                 输出
─────────          ──────────           ─────────
RGB (2帧)     →   CLIP ViT-B/32     →  视觉 token
                   (224×224, 随机裁剪 + 色彩抖动)

Depth (1帧)   →   同一 ViT-B/32架构  →  深度 token
                   (复制到3通道, 灰度, clamp 0.5m)

Left CoinFT   →   CausalConv        →  左力 token
(32帧 F/T)         (因果卷积网络)

Right CoinFT  →   CausalConv        →  右力 token
(32帧 F/T)         (因果卷积网络)

          ↓ 所有 token ↓
   ┌─────────────────────────┐
   │ Transformer Self-Attn   │  ← 跨模态融合
   │ (一层 Encoder)          │
   └──────────┬──────────────┘
              ↓
   拼接 本体感知（末端位姿, 2帧）
              ↓
   组合特征向量 → 条件输入扩散策略
```

#### 设计细节

- **RGB**：取最近 2 个时间步的图像，经 CLIP 预训练 ViT-B/32 编码，数据增强包括随机裁剪和色彩抖动
- **Depth**：复制到 3 通道以兼容 RGB ViT 架构；截断到 0.5m 以强调近距离几何，抑制远处背景干扰
- **F/T**：每颗 CoinFT 取最近 32 帧（约 0.09s @ 360Hz）的力/力矩历史，通过因果卷积网络（CausalConv）压缩为单个特征向量。使用因果卷积而非双向卷积，是因为只有过去的力信息可用
- **Self-Attention 融合**：将视觉 token、深度 token、左力 token、右力 token 拼接后送入 Transformer Encoder 层做自注意力，让模型自主学习跨模态的关联（如"看到接触 + 力信号上升 → 识别为有效接触"）
- **本体感知**：末端执行器位姿（2 帧），作为低维信号直接拼接到融合特征后，不经过 Transformer

### 4. 扩散策略（Diffusion Policy）

#### 输入 → 输出

```
输入：观测编码模块的组合特征向量（条件信号）
输出：21D 动作向量
```

#### 21D 动作向量分解

| 分量 | 维度 | 格式 |
|------|------|------|
| Reference Pose | 9D | 3D 位置 + 6D 连续旋转 |
| Virtual Target Pose | 9D | 同上 |
| Stiffness | 1D | 标量 |
| Gripper Width | 1D | 标量 |
| Grasp Force | 1D | 标量 |

#### 训练前的标签生成

扩散策略的训练标签不是直接标注的，而是从示教数据**后处理**得到的：

1. **Reference Pose & Virtual Target Pose & Stiffness**：按照 ACP 框架，对示教过程中记录的力-运动数据进行后处理，拟合时变 3D 虚拟弹簧模型，反解出参考位姿、虚拟目标位姿和刚度标签
2. **Grasp Force**：直接从两颗 CoinFT 的读数中提取（取两指在夹持轴方向的平均值）
3. **Gripper Width**：从手指间距的运动学测量中获得

#### 推理时的执行流程

```
扩散去噪（多步迭代）
       ↓
  21D 动作向量
       ↓
  ┌────┴─────────────────────────────────┐
  ↓                                       ↓
Ref Pose + VT Pose + Stiffness      Gripper Width + Grasp Force
  ↓                                       ↓
重建完整刚度矩阵（按 ACP）           发送给夹持力控制器
  ↓
刚度矩阵 + VT Pose → 手腕柔顺控制器
```

### 3. 夹爪层：速度-力混合导纳控制 @ 30Hz

#### 抓取力测量

抓取力 $f_G$ 定义为两个 CoinFT 在**夹持方向**上读数的平均值：

$$f_G = \frac{1}{2}(|W_L[0]| + |W_R[0]|)$$

> 这里取法向力分量（抓取轴方向），反映夹爪对物体的实际夹持力大小。

#### 控制律

$$v_G = K_p(x_G^* - x_G) + K_f(f_G^* - f_G)$$

| 符号 | 含义 |
| ------ | ------ |
| $v_G$ | 夹爪速度指令（输出给夹爪电机） |
| $x_G^*$ | 策略输出的**期望夹爪宽度** |
| $x_G$ | 当前实际夹爪宽度 |
| $K_p$ | 位置增益：控制宽度跟踪的灵敏度 |
| $f_G^*$ | 策略输出的**期望抓取力** |
| $f_G$ | CoinFT 实测的当前抓取力 |
| $K_f$ | 力增益：控制力跟踪的灵敏度 |

**直觉理解**：这是一个**位置-力混合控制器**：

- 第一项 $K_p(x_G^* - x_G)$：让夹爪趋向目标宽度（位置环）
- 第二项 $K_f(f_G^* - f_G)$：让抓取力趋向目标力（力环）
- 两项共同作用：夹住物体后，即使物体形状/柔软度不同，也能维持恒定的抓取力

#### 实际效果

以**穿刺西葫芦任务**为例：

- 策略输出一个合适的 $f_G^*$，确保夹持足够紧不滑落
- 穿刺过程中西葫芦受力变形，夹爪宽度被动变化
- 力控制器以 30Hz 持续补偿，维持恒定夹持力
- **没有力控制的 baseline (DP)**：夹爪只跟踪宽度，穿刺时西葫芦频繁滑落（成功率 30% vs 80%）

### 4. 为什么选择导纳控制而非阻抗控制？

论文选用的是**导纳控制（Admittance Control）**而非阻抗控制（Impedance Control），这一选择与硬件特性密切相关：

| 特性 | 导纳控制 | 阻抗控制 |
| ------ | --------- | --------- |
| 输入 | 力 → 输出位置/速度 | 位置偏差 → 输出力 |
| 适用机器人 | **位置控制型**（如 UR5e） | 力矩控制型（如 Franka） |
| 需要 | 外部力传感器（CoinFT ✓） | 关节力矩传感器 |
| UMI-FT 选择理由 | UR5e 是位控机器人，CoinFT 提供外部力测量 | — |

> UR5e 本身是位置控制型机器人，不能直接输出关节力矩。导纳控制将测量到的外力转换为位置修正量，完美匹配 UR5e 的控制接口。

### 5. 整体信息流总结

```text
Diffusion Policy (1Hz)
  │
  ├─→ Reference Pose (9D)  ─────────────┐
  ├─→ Virtual Target Pose (9D) ─────────┤
  ├─→ Stiffness (1D) ──→ 重建 6×6 K ──→├──→ Wrist Admittance Controller (500Hz)
  │                                     │      输入: Virtual Target, K, W_wrist
  │   CoinFT L/R (360Hz) ──→ 坐标变换 ──┘      输出: 末端位置修正 → UR5e 关节指令
  │     │
  │     └──→ 抓取力 f_G ──────────────────┐
  ├─→ Gripper Width (1D) ───────────────┤──→ Grasp Force Controller (30Hz)
  └─→ Grasp Force (1D) ────────────────┘      输出: 夹爪速度 v_G → WSG50 指令
```

> **核心洞察**：Diffusion Policy 的 21D 输出不是直接的电机指令，而是底层控制器的**参数化配置**——它告诉控制器「目标在哪」「该多柔顺」「该夹多紧」，控制器在高频闭环中自主解决力的实时调节。这种**策略-控制器分离的架构**，是 UMI-FT 能在真实世界安全执行力敏感任务的根本保障。

### 5. 手腕柔顺控制器（Wrist Compliance Controller）

#### 输入 → 输出

```
输入：
  - Virtual Target Pose（来自策略）
  - 刚度矩阵 K（由策略输出的 stiffness 标量重建）
  - 双指合力反馈 W_wrist = Ad^T_{S1,T} · W_{S1} + Ad^T_{S2,T} · W_{S2}

输出：
  - 手臂关节速度指令
```

#### 设计细节

- **控制方式**：6D 任务空间导纳控制（Admittance Control），将机器人手臂模拟为虚拟弹簧-质量-阻尼系统
- **力反馈来源**：两颗指尖 CoinFT 的测量值 W_{S1}、W_{S2}，通过伴随变换矩阵 (Adjoint Transformation) 转换到工具坐标系后求和
- **工具中心点 (TCP)**：设置在两指尖中心（而非法兰），使擦白板等任务的柔顺行为更自然
- **运行频率**：500 Hz（机器人硬件极限）
- **柔顺维度**：论文中实验仅在 3D 平移空间启用柔顺，但框架天然支持扩展到完整 6D

### 6. 夹持力控制器（Grasp Force Controller）

#### 输入 → 输出

```
输入：
  - 目标夹持力 f*_G（来自策略）
  - 目标夹爪宽度 x*_G（来自策略）
  - 实际夹持力 f_G = (|W_{S1}| + |W_{S2}|) / 2（CoinFT 夹持轴读数均值）
  - 实际夹爪位置 x_G

输出：
  - 夹爪速度指令 v_G
```

#### 控制律

```
v_G = K_p · (x*_G - x_G) + K_f · (f*_G - f_G)
```

- **K_p**（位置增益）：控制夹爪的位置跟踪灵敏度
- **K_f**（力增益）：控制夹持力的调节灵敏度
- 两者可调节以平衡位置跟踪与力跟踪的权重

#### 设计要点

- **运行频率**：30 Hz（夹爪硬件极限）
- **融合位置与力控制**：在自由空间（无接触）时，位置项主导夹爪运动；接触后，力项自动接管，实现平滑过渡
- **安全阈值**：运行时通过 CoinFT 实时监测力，超过安全阈值（如夹持方向 25N、其他方向 20N）自动停机
