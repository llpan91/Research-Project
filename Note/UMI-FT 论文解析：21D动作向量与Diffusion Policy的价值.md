# UMI-FT 论文解析：21D 动作向量与 Diffusion Policy 的实际价值

> 论文：*In-the-Wild Compliant Manipulation with UMI-FT*
> arXiv: 2601.09988v1, 15 Jan 2026

---

## 一、为什么输出 21 维动作向量（含 9D 位姿），而不是 6DoF？

### 1. 21 维向量的具体构成

论文 Section IV-B (Output Decoding) 明确给出了分解：

| 分量 | 维度 | 说明 |
|------|------|------|
| Reference Pose | **9D** | 3D 平移 + 6D 旋转（旋转矩阵前两行） |
| Virtual Target Pose | **9D** | 同上格式，柔顺控制器的实际设定目标 |
| Stiffness Value | **1D** | 标量，编码机械臂刚度矩阵 |
| Gripper Width | **1D** | 期望夹爪宽度 |
| Grasp Force | **1D** | 期望夹持力 |
| **总计** | **21D** | |

---

### 2. 为什么旋转用 6D 而不是 3D（欧拉角）或 4D（四元数）？

这是遵循 Diffusion Policy [30] 的惯例，其数学根源来自 Zhou et al. 的连续旋转表示理论：

- **欧拉角 (3D)**：存在**万向节锁 (Gimbal Lock)**，且角度空间存在 ±π 跳变等不连续性，对神经网络的回归学习非常不友好
- **四元数 (4D)**：存在**对径等价问题** —— `q` 和 `-q` 表示同一旋转，导致损失函数的拓扑空间不连续，网络学习时会在两个等价点间震荡
- **旋转矩阵前两行 (6D)**：这是一种**拓扑连续的表示**。取 3×3 旋转矩阵的前两行（6 个数），推理时通过 Gram-Schmidt 正交化恢复第三行，重建完整旋转矩阵。这种表示在 SO(3) 到 ℝ⁶ 的映射上是连续的，对神经网络回归最友好

> **结论**：9D = 3D position + 6D continuous rotation，本质上就是 6DoF 位姿的一种**对学习更友好的参数化形式**，不是增加了新的信息维度。

---

### 3. 为什么要输出「两个」位姿（Reference Pose + Virtual Target Pose）？

这是 UMI-FT 最核心的设计之一，来源于 ACP (Adaptive Compliance Policy) [8] 框架。两个位姿配合 stiffness 标量，共同编码了一个**时变虚拟弹簧模型**：

```
                   Stiffness (k)
Reference Pose ←────/\/\/\/────→ Virtual Target Pose
  (弹簧一端)           弹簧          (弹簧另一端 = 控制器设定目标)
```

- **Reference Pose（参考位姿）**：策略认为末端执行器「应该在的位置」，即名义轨迹点
- **Virtual Target Pose（虚拟目标位姿）**：发送给底层柔顺控制器的**实际设定目标**，它与参考位姿之间的偏移量编码了期望施加的力
- **Stiffness（刚度）**：弹簧常数 `k`，决定偏移量到力的映射关系

**核心逻辑**：

```
F = k × (Virtual Target - Reference Pose)
```

**具体例子（白板擦除任务）**：

- 策略希望擦板时施加 10N 向下力
- 不直接输出"10N"，而是输出 Reference Pose 在白板表面、Virtual Target 穿透白板表面几毫米、Stiffness 设为合适的值
- 底层柔顺控制器追踪 Virtual Target，遇到白板阻挡后自然产生弹簧力，实现柔顺接触

**为什么不直接输出力？**

因为直接预测力并执行需要精确的力控制器，且纯力控制在自由空间（未接触时）没有意义。用虚拟弹簧模型，**自由空间走位置、接触后走力**，天然统一了两种行为模式。

## 二、Diffusion Policy 在这个工作中的实际价值

### 1. 系统架构中的定位

论文 Fig. 3 展示了三层控制回路：

```
┌────────────────────────────────────────────────────────────┐
│  Learned Visuomotor Policy (Diffusion Policy)    @ 1 Hz    │  ← 最慢，做"决策"
│  输入: RGB, Depth, F/T (32帧), Proprioception              │
│  输出: 21D 动作向量                                          │
├────────────────────────────────────────────────────────────┤
│  Arm Compliance Controller (Admittance)          @ 500 Hz  │  ← 中层，做"柔顺执行"
│  Grasp Force Controller                          @ 30 Hz   │  ← 底层，做"夹持力调节"
└────────────────────────────────────────────────────────────┘
```

Diffusion Policy 是**最上层的「大脑」**，以 1Hz 的频率生成包含位姿、刚度、夹持力的完整动作指令；底层两个基于模型的控制器以高频率做实际的物理交互。

---

### 2. 为什么必须用 Diffusion Policy 而非普通回归网络？

#### (a) 动作的多模态性（Multi-modality）

在灯泡插入任务中，当针脚需要对齐插槽时，机器人可以顺时针或逆时针旋转搜索——两种方向都是有效的。普通回归网络（MSE loss）会把两个模态取平均，输出一个**介于两者之间的无效动作**（比如不旋转）。Diffusion Policy 通过去噪过程对完整的动作分布建模，可以**采样到任意一个有效模态**而不会模态坍缩。

#### (b) 时序动作块预测（Action Chunk Prediction）

Diffusion Policy 不是逐帧预测单个动作，而是一次生成**一段未来动作序列**。这对柔顺操作至关重要：

- 擦白板时需要持续稳定的接触力轨迹
- 穿刺西葫芦时需要平滑的力递增过程
- 时序一致的动作块天然避免了帧间抖动

#### (c) 灵活的多模态条件输入

Diffusion Policy 本质是条件生成模型，可以自然地接受**任意多模态输入**作为条件。论文中的观测编码流程：

```
RGB (2帧)          → ViT-B/32      ──┐
Depth              → ViT-B/32      ──┤
Left F/T (32帧)    → CausalConv    ──┤→ Self-Attention Fusion → Diffusion → 21D Action
Right F/T (32帧)   → CausalConv    ──┤
Proprioception     ─────────────────┘
```

各个模态的 token 经过 Self-Attention 融合后，作为条件送入扩散过程。这种架构天然支持 UMI-FT 新增的力/力矩模态，不需要重新设计网络结构。

---

### 3. 实验中的直接证据

论文所有 baseline 对比中，Diffusion Policy 都是**固定的策略骨干**，变量是输入模态和输出执行方式：

| 方法 | 策略骨干 | 力观测 | 柔顺控制 | 白板擦除 | 灯泡插入 |
|------|---------|--------|---------|---------|---------|
| **Ours (ACP)** | Diffusion Policy | CoinFT | 有 | **92%** | **95%** |
| DP w/ F | Diffusion Policy | CoinFT | 无 | 28% | 60% |
| DP | Diffusion Policy | 无 | 无 | 16% | 0% |
| DP w/ CM | Diffusion Policy | 接触麦克风 | 无 | 0% | 20% |

- 即使在没有柔顺控制的 baseline (DP w/ F) 中，Diffusion Policy + 力观测已经能完成部分任务（灯泡 60%、西葫芦 70%），说明 Diffusion Policy 本身确实学到了**力感知的操作策略**
- 完整系统 (ACP) 的大幅提升来自**将 Diffusion Policy 的高层决策与底层柔顺控制解耦**

---

### 4. 总结：Diffusion Policy 的不可替代价值

| 价值 | 具体体现 |
|------|---------|
| **多模态动作分布建模** | 避免接触丰富任务中的模态平均问题 |
| **时序一致的动作块** | 保证力轨迹平滑，避免抖动 |
| **灵活的多模态条件输入** | 天然融合 RGB + Depth + F/T + Proprioception |
| **高层-低层解耦** | 1Hz 做决策 + 500Hz 做柔顺执行，既有智能又有反应速度 |

> Diffusion Policy 在这个工作中不是「可以替换的模块」，而是整个系统的**决策核心**——它以低频率输出「做什么」（21D 向量），底层控制器以高频率解决「怎么做」（柔顺接触）。

## 三、底层控制器如何实现高频柔顺接触

Diffusion Policy 以 ~1Hz 输出 21D 动作向量后，**真正的物理交互由两个基于模型的高频控制器完成**。这是整个系统能安全、稳定地完成力敏感任务的关键。

### 1. 总体架构：三环并行、频率分层

```
┌─────────────────────────────────────────────────────────────────┐
│ 感知输入                                                          │
│ RGB, Depth, Left/Right CoinFT (32帧), Proprioception            │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第 1 层：Visuomotor Policy（Diffusion Policy）       @ ~1 Hz     │
│ 输出: Ref Pose, Virtual Target, Stiffness, Gripper Width, Force │
└────────┬──────────────────────────────────┬─────────────────────┘
         ▼                                  ▼
┌──────────────────────────┐  ┌────────────────────────────────┐
│ 第 2 层：Wrist Compliance │  │ 第 3 层：Grasp Force Controller │
│ Controller (手臂导纳控制)  │  │ (夹爪力控制)                     │
│ @ 500 Hz                  │  │ @ 30 Hz                         │
│ 输入: Virtual Target,     │  │ 输入: 目标抓取力 f_G*,           │
│   Stiffness, F/T Wrench  │  │   CoinFT 实测抓取力 f_G          │
│ 输出: 机械臂关节指令        │  │ 输出: 夹爪速度指令               │
└──────────────────────────┘  └────────────────────────────────┘
```

**关键设计思想**：策略做低频「决策」，控制器做高频「反应」。这种分层使得：

- 策略可以从容处理高维视觉输入（1Hz 足够）
- 物理接触的力反馈在 500Hz 内闭环，远快于物体动力学变化
- 两层完全解耦，底层控制器不需要任何学习

---

### 2. 手臂层：6D 导纳控制（Admittance Control）@ 500Hz

#### 核心原理

导纳控制将机械臂末端模拟为**虚拟弹簧-质量-阻尼器（Spring-Mass-Damper）系统**：

```
外部环境力 F_ext
       │
       ▼
  ┌─────────┐     弹簧(K)      ┌───────────┐
  │ 末端执行器 │←──/\/\/\/──────│ Virtual    │
  │ (实际位置) │←──[阻尼器]─────│ Target Pose│
  │          │     阻尼(D)      │ (设定目标)  │
  └─────────┘                  └───────────┘
```

当末端执行器接触环境产生外力时，导纳控制器让末端执行器**像弹簧一样柔顺退让**，而不是刚性地强制到达目标位置。

#### 力信号融合

两个 CoinFT 传感器的 6D 力/力矩测量需要先统一到机器人工具坐标系：

$$W_{wrist} = \text{Ad}_{S_1T}^T \cdot W_{S1} + \text{Ad}_{S_2T}^T \cdot W_{S2}$$

其中：

- $W_{S1}, W_{S2}$：左右 CoinFT 的 6D 力/力矩读数（各 $\in \mathbb{R}^6$）
- $\text{Ad}_{S_iT}^T$：传感器坐标系到工具坐标系的**伴随变换矩阵的转置**（用于力旋量的坐标变换）
- $W_{wrist}$：合并后的工具坐标系力/力矩，作为导纳控制器的反馈输入

> **为什么要合并两个传感器？** 因为外力可能作用在任一手指上（如擦白板时主要是一侧接触），合并后控制器能响应任意手指上的外力。

#### Diffusion Policy 输出如何驱动导纳控制器

策略输出的三个关键量直接控制导纳行为：

| 策略输出 | 导纳控制器中的角色 | 物理含义 |
| --------- | ------------------ | --------- |
| **Virtual Target Pose** | 控制器的**设定目标位置** | 弹簧的平衡点，末端被拉向此处 |
| **Reference Pose** | 用于计算期望力 | 与 Virtual Target 的偏差编码期望施力方向和大小 |
| **Stiffness 标量** | 重建为完整的 **6×6 刚度矩阵 K** | 弹簧软硬程度；高刚度 → 位置跟踪精确；低刚度 → 柔顺退让 |

**刚度矩阵的重建过程**（遵循 ACP [8] 方法）：

- 策略只输出 1 个标量而非完整 6×6 矩阵
- 训练前通过对示教数据中的力-运动关系进行后处理，用**时变 3D 弹簧模型**拟合得到刚度和虚拟目标的标签
- 推理时，将标量重建为完整刚度矩阵，送入导纳控制器

#### 不同场景下的刚度自适应

这是 "Adaptive" Compliance Policy 名称的由来——**策略学会了根据任务阶段动态调整刚度**：

```
灯泡插入任务的刚度变化（示意）：

接近阶段:  K = 高  → 精确位置跟踪，快速接近插座
搜索阶段:  K = 低  → 柔顺探索，bayonet 针脚碰到插槽边缘时不会卡死
插入阶段:  K = 中  → 适度力施加，压缩弹簧电极
旋转阶段:  K = 低  → 柔顺旋转，保持针脚在槽内接触
```

### 3. 夹爪层：速度-力混合导纳控制 @ 30Hz

#### 抓取力测量

抓取力 $f_G$ 定义为两个 CoinFT 在**夹持方向**上读数的平均值：

$$f_G = \frac{1}{2}(|W_L[0]| + |W_R[0]|)$$

> 这里取法向力分量（抓取轴方向），反映夹爪对物体的实际夹持力大小。

#### 控制律

$$v_G = K_p(x_G^* - x_G) + K_f(f_G^* - f_G)$$

| 符号 | 含义 |
| ------ | ------ |
| $v_G$ | 夹爪速度指令（输出给夹爪电机） |
| $x_G^*$ | 策略输出的**期望夹爪宽度** |
| $x_G$ | 当前实际夹爪宽度 |
| $K_p$ | 位置增益：控制宽度跟踪的灵敏度 |
| $f_G^*$ | 策略输出的**期望抓取力** |
| $f_G$ | CoinFT 实测的当前抓取力 |
| $K_f$ | 力增益：控制力跟踪的灵敏度 |

**直觉理解**：这是一个**位置-力混合控制器**：

- 第一项 $K_p(x_G^* - x_G)$：让夹爪趋向目标宽度（位置环）
- 第二项 $K_f(f_G^* - f_G)$：让抓取力趋向目标力（力环）
- 两项共同作用：夹住物体后，即使物体形状/柔软度不同，也能维持恒定的抓取力

#### 实际效果

以**穿刺西葫芦任务**为例：

- 策略输出一个合适的 $f_G^*$，确保夹持足够紧不滑落
- 穿刺过程中西葫芦受力变形，夹爪宽度被动变化
- 力控制器以 30Hz 持续补偿，维持恒定夹持力
- **没有力控制的 baseline (DP)**：夹爪只跟踪宽度，穿刺时西葫芦频繁滑落（成功率 30% vs 80%）

### 4. 为什么选择导纳控制而非阻抗控制？

论文选用的是**导纳控制（Admittance Control）**而非阻抗控制（Impedance Control），这一选择与硬件特性密切相关：

| 特性 | 导纳控制 | 阻抗控制 |
| ------ | --------- | --------- |
| 输入 | 力 → 输出位置/速度 | 位置偏差 → 输出力 |
| 适用机器人 | **位置控制型**（如 UR5e） | 力矩控制型（如 Franka） |
| 需要 | 外部力传感器（CoinFT ✓） | 关节力矩传感器 |
| UMI-FT 选择理由 | UR5e 是位控机器人，CoinFT 提供外部力测量 | — |

> UR5e 本身是位置控制型机器人，不能直接输出关节力矩。导纳控制将测量到的外力转换为位置修正量，完美匹配 UR5e 的控制接口。

### 5. 整体信息流总结

```text
Diffusion Policy (1Hz)
  │
  ├─→ Reference Pose (9D)  ─────────────┐
  ├─→ Virtual Target Pose (9D) ─────────┤
  ├─→ Stiffness (1D) ──→ 重建 6×6 K ──→├──→ Wrist Admittance Controller (500Hz)
  │                                     │      输入: Virtual Target, K, W_wrist
  │   CoinFT L/R (360Hz) ──→ 坐标变换 ──┘      输出: 末端位置修正 → UR5e 关节指令
  │     │
  │     └──→ 抓取力 f_G ──────────────────┐
  ├─→ Gripper Width (1D) ───────────────┤──→ Grasp Force Controller (30Hz)
  └─→ Grasp Force (1D) ────────────────┘      输出: 夹爪速度 v_G → WSG50 指令
```

> **核心洞察**：Diffusion Policy 的 21D 输出不是直接的电机指令，而是底层控制器的**参数化配置**——它告诉控制器「目标在哪」「该多柔顺」「该夹多紧」，控制器在高频闭环中自主解决力的实时调节。这种**策略-控制器分离的架构**，是 UMI-FT 能在真实世界安全执行力敏感任务的根本保障。
