## 1. 什么是扩散模型

### 1.1 直觉理解

扩散模型的核心思想可以用一个简单的比喻来理解：想象你把一滴墨水滴入一杯清水中，墨水会逐渐扩散直到均匀分布在整杯水中——这就是**前向扩散过程（Forward Diffusion）**。而扩散模型要做的，是学会**逆向这个过程**：从一杯均匀的墨水中，逐步"还原"出最初那滴墨水的形状。

在数学上，扩散模型是一类**基于似然的生成模型（Likelihood-based Generative Model）**，通过两个互逆的马尔可夫链过程来实现数据生成：

- **前向过程（Forward Process / Diffusion Process）**：逐步向数据中添加高斯噪声，直到数据变为纯噪声
- **逆向过程（Reverse Process / Denoising Process）**：学习从噪声中逐步去噪，恢复出原始数据

### 1.2 数学表述

#### 前向过程

给定原始数据 $x_0 \sim q(x_0)$，前向过程定义为一个马尔可夫链，在每个时间步 $t$ 添加少量高斯噪声：

$$q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t} \, x_{t-1}, \beta_t \mathbf{I})$$

其中 $\beta_t \in (0, 1)$ 是预先设定的噪声调度系数（noise schedule），控制每一步添加噪声的强度。

利用重参数化技巧，可以直接从 $x_0$ 得到任意时间步 $t$ 的 $x_t$：

$$q(x_t | x_0) = \mathcal{N}(x_t; \sqrt{\bar{\alpha}_t} \, x_0, (1 - \bar{\alpha}_t) \mathbf{I})$$

其中 $\alpha_t = 1 - \beta_t$，$\bar{\alpha}_t = \prod_{s=1}^{t} \alpha_s$。

当 $T \to \infty$ 时，$x_T$ 趋近于标准高斯分布 $\mathcal{N}(0, \mathbf{I})$。

#### 逆向过程

逆向过程同样定义为一个马尔可夫链，由神经网络参数化：

$$p_\theta(x_{t-1} | x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \Sigma_\theta(x_t, t))$$

$\color{red}{模型学习预测每一步去噪的均值}$ $\color{red}{\mu_\theta}$ $\color{red}{和方差}$ $\color{red}{\Sigma_\theta}$ $\color{red}{。}$
在实践中，方差通常固定或简单参数化，模型主要学习预测以下三种等价目标之一：

| 预测目标 | 含义 | 代表工作 |
|---------|------|---------|
| $\epsilon_\theta(x_t, t)$ | 预测添加的噪声 | DDPM |
| $x_\theta(x_t, t)$ | 预测原始数据 | 部分扩散模型变体 |
| $v_\theta(x_t, t)$ | 预测速度（velocity） | Progressive Distillation |

#### 训练目标

训练通过最大化变分下界（ELBO）实现，简化后的损失函数为：

$$L_{\text{simple}} = \mathbb{E}_{t, x_0, \epsilon} \left[ \| \epsilon - \epsilon_\theta(\sqrt{\bar{\alpha}_t} \, x_0 + \sqrt{1 - \bar{\alpha}_t} \, \epsilon, \, t) \|^2 \right]$$

即在随机时间步 $t$，向数据 $x_0$ 添加随机噪声 $\epsilon$，训练网络预测这个噪声。

### 1.3 采样过程

推理（生成）时，从 $x_T \sim \mathcal{N}(0, \mathbf{I})$ 出发，迭代执行去噪步骤：

$$x_{t-1} = \frac{1}{\sqrt{\alpha_t}} \left( x_t - \frac{1 - \alpha_t}{\sqrt{1 - \bar{\alpha}_t}} \epsilon_\theta(x_t, t) \right) + \sigma_t z, \quad z \sim \mathcal{N}(0, \mathbf{I})$$

经过 $T$ 步迭代，最终得到生成的样本 $x_0$。

## 2. 扩散模型与传统神经网络模型的关系

### 2.1 生成模型的谱系

扩散模型属于**深度生成模型（Deep Generative Models）**家族的一员。下面将其与其他经典生成模型进行系统比较。

```
                        深度生成模型
                            │
            ┌───────────────┼───────────────────┐
            │               │                   │
        显式密度          隐式密度             其他
            │               │                   │
    ┌───────┼───────┐       │            ┌──────┼──────┐
    │       │       │       │            │      │      │
   VAE    Flow   扩散模型   GAN       Autoregressive  EBM
                                      Models
```

### 2.2 与 GAN（生成对抗网络）的对比

| 维度 | GAN | 扩散模型 |
|------|-----|---------|
| **架构** | 生成器 + 判别器，对抗训练 | 单一去噪网络，去噪训练 |
| **训练稳定性** | 不稳定，模式崩溃常见 | 稳定，损失函数为简单的 MSE |
| **生成质量** | 高质量但多样性受限 | 高质量且高多样性 |
| **生成速度** | 单次前向传播，极快 | 需多步迭代去噪，较慢 |
| **模式覆盖** | 容易模式崩溃（mode collapse） | 天然覆盖完整数据分布 |
| **似然估计** | 无法估计 | 可估计变分下界 |
| **可控生成** | 需额外设计（如 cGAN） | 天然支持条件生成和引导 |

**关键洞察**：GAN 通过对抗博弈隐式学习数据分布，而扩散模型通过逐步去噪显式建模数据分布的逆过程。扩散模型牺牲了生成速度，换来了训练稳定性和生成多样性的显著提升。

### 2.3 与 VAE（变分自编码器）的对比

| 维度 | VAE | 扩散模型 |
|------|-----|---------|
| **编码器** | 可学习的编码器将数据映射到潜空间 | 前向过程是固定的加噪过程（无需学习） |
| **潜空间** | 单一潜变量 $z$ | $T$ 个中间潜变量 $x_1, ..., x_T$ |
| **解码** | 一步解码 | 多步迭代去噪 |
| **损失函数** | 重构损失 + KL 散度 | 简化的去噪损失 |
| **生成质量** | 受限于后验近似，常模糊 | 高质量，细节丰富 |
| **潜空间结构** | 连续且有结构，适合插值 | 中间步无显式结构 |

**关键洞察**：VAE 和扩散模型都基于变分推断框架。VAE 使用可学习的编码器一步映射到潜空间，而扩散模型的"编码器"是固定的多步加噪过程。扩散模型可以看作是一个**层次化的 VAE（Hierarchical VAE）**，具有 $T$ 层潜变量，且编码器固定。

### 2.4 与 Normalizing Flows 的对比

| 维度 | Normalizing Flows | 扩散模型 |
|------|-------------------|---------|
| **变换** | 可逆变换，精确似然 | 前向/逆向不严格可逆 |
| **架构约束** | 必须使用可逆网络（如 RealNVP） | 无架构约束，可用任意网络 |
| **似然计算** | 精确计算 | 变分下界近似 |
| **表达能力** | 受限于可逆性约束 | 灵活，表达能力强 |

### 2.5 与 Autoregressive Models 的对比

| 维度 | Autoregressive | 扩散模型 |
|------|----------------|---------|
| **生成顺序** | 按序列顺序逐元素生成 | 全局并行去噪（所有像素同时处理） |
| **并行性** | 训练可并行，生成不可并行 | 训练可并行，生成需多步但每步全局并行 |
| **适用数据** | 序列数据（文本、音频） | 连续数据（图像、轨迹） |
| **似然计算** | 精确计算 | 变分下界近似 |

### 2.6 与 Score-based Models 的统一

扩散模型与**基于分数的生成模型（Score-based Generative Models）**在数学上是等价的。Song et al. (2021) 通过**随机微分方程（SDE）**框架统一了两者：

- **前向 SDE**：$dx = f(x, t) \, dt + g(t) \, dw$
- **逆向 SDE**：$dx = [f(x,t) - g(t)^2 \nabla_x \log p_t(x)] \, dt + g(t) \, d\bar{w}$

其中 $\nabla_x \log p_t(x)$ 即为分数函数（score function）。去噪扩散模型训练的噪声预测网络 $\epsilon_\theta$ 本质上就是在估计分数函数。

### 2.7 扩散模型中使用的骨干网络

扩散模型本身**不是一种新的网络架构**，而是一种新的**<span style="color:red">训练范式和生成框架</span>**。它依赖传统神经网络作为去噪网络的骨干：

| 骨干网络 | 应用场景 | 代表工作 |
|---------|---------|---------|
| **U-Net** | 图像生成（最经典） | DDPM, Stable Diffusion |
| **Transformer (DiT)** | 大规模图像/视频生成 | DiT, Sora |
| **1D U-Net / MLP** | 轨迹/动作生成 | Diffusion Policy |
| **3D U-Net** | 视频生成 | Video Diffusion Models |
| **Graph Neural Network** | 分子生成 | GeoDiff |

## 3. 经典算法与核心思路

### 3.1 DDPM（Denoising Diffusion Probabilistic Models, 2020）

**论文**：Ho et al., "Denoising Diffusion Probabilistic Models", NeurIPS 2020

**核心贡献**：

- 首次证明扩散模型可以生成与 GAN 媲美的高质量图像
- 提出简化的训练目标 $L_{\text{simple}}$，预测噪声而非直接预测均值
- 建立了扩散模型与去噪分数匹配（denoising score matching）的联系

**算法流程**：

```
训练阶段：
1. 从数据集采样 x_0
2. 随机采样时间步 t ~ Uniform({1, ..., T})
3. 随机采样噪声 ε ~ N(0, I)
4. 计算 x_t = sqrt(ᾱ_t) * x_0 + sqrt(1 - ᾱ_t) * ε
5. 最小化 ||ε - ε_θ(x_t, t)||²

采样阶段：
1. 采样 x_T ~ N(0, I)
2. for t = T, T-1, ..., 1:
     预测 ε_θ(x_t, t)
     计算 x_{t-1}（去噪一步）
3. 返回 x_0
```

**局限性**：需要 $T=1000$ 步采样，生成速度很慢。

### 3.2 DDIM（Denoising Diffusion Implicit Models, 2021）

**论文**：Song et al., "Denoising Diffusion Implicit Models", ICLR 2021

**核心思路**：将 DDPM 的随机马尔可夫链采样推广为**非马尔可夫、确定性**的采样过程。

**关键公式**：

$$x_{t-1} = \sqrt{\bar{\alpha}_{t-1}} \underbrace{\left( \frac{x_t - \sqrt{1 - \bar{\alpha}_t} \, \epsilon_\theta(x_t, t)}{\sqrt{\bar{\alpha}_t}} \right)}_{\text{预测的 } x_0} + \sqrt{1 - \bar{\alpha}_{t-1} - \sigma_t^2} \cdot \epsilon_\theta(x_t, t) + \sigma_t \epsilon$$

- 当 $\sigma_t = 0$ 时，采样过程完全确定性（DDIM）
- 当 $\sigma_t = \sqrt{\frac{1-\bar{\alpha}_{t-1}}{1-\bar{\alpha}_t}} \sqrt{1 - \frac{\bar{\alpha}_t}{\bar{\alpha}_{t-1}}}$ 时，等价于 DDPM

**核心优势**：

- 可以跳步采样（如从 1000 步减少到 50 步甚至 10 步），极大加速生成
- 确定性采样使得潜空间有意义，支持插值操作
- 训练好的 DDPM 模型可以直接用 DDIM 采样，**无需重新训练**

### 3.3 Classifier-Free Guidance（无分类器引导, 2022）

**论文**：Ho & Salimans, "Classifier-Free Diffusion Guidance", 2022

**核心思路**：在不需要额外分类器的情况下实现条件生成的引导。

**方法**：

- 训练时，随机以一定概率丢弃条件信息 $c$（用空条件 $\varnothing$ 替代），使模型同时学习条件生成和无条件生成
- 推理时，通过线性外推增强条件引导的强度：

$$\tilde{\epsilon}_\theta(x_t, c) = (1 + w) \cdot \epsilon_\theta(x_t, c) - w \cdot \epsilon_\theta(x_t, \varnothing)$$

其中 $w$ 是引导强度（guidance scale）。$w > 0$ 时，模型会更强烈地朝条件方向生成。

**影响**：这一技术已成为几乎所有条件扩散模型的标配，是 Stable Diffusion、DALL-E 等文生图模型的核心组件。

### 3.4 Latent Diffusion Models / Stable Diffusion（2022）

**论文**：Rombach et al., "High-Resolution Image Synthesis with Latent Diffusion Models", CVPR 2022

**核心思路**：不在像素空间运行扩散过程，而在**预训练自编码器的潜空间**中运行。

**架构**：

```
输入图像 x (512×512×3)
    │
    ▼
[预训练 Encoder E] ──→ 潜表示 z (64×64×4)
                            │
                            ▼
                    [扩散过程在潜空间进行]
                    [U-Net + Cross-Attention]
                    [条件输入: CLIP text embedding]
                            │
                            ▼
                    去噪后的 z₀ (64×64×4)
                            │
                            ▼
                    [预训练 Decoder D] ──→ 生成图像 (512×512×3)
```

**核心优势**：

- 计算量降低约 4-16 倍（取决于压缩比）
- 潜空间更紧凑，去除了感知上冗余的信息
- 可灵活接入各种条件（文本、图像、语义图等）通过 cross-attention

### 3.5 Score SDE（基于随机微分方程的扩散模型, 2021）

**论文**：Song et al., "Score-Based Generative Modeling through Stochastic Differential Equations", ICLR 2021

**核心思路**：用连续时间的 SDE 统一离散的 DDPM 和连续的 NCSN（Noise Conditional Score Network）。

**框架**：

- 将离散的加噪/去噪过程推广为连续的随机微分方程
- 前向 SDE 描述数据如何逐渐变成噪声
- 逆向 SDE（或对应的概率流 ODE）描述如何从噪声恢复数据
- 不同的 SDE 选择对应不同的扩散模型变体

**统一视角**：

| SDE 类型 | 对应模型 |
|---------|---------|
| VP-SDE (Variance Preserving) | DDPM |
| VE-SDE (Variance Exploding) | NCSN / SMLD |
| sub-VP SDE | 改进的 DDPM |

### 3.6 Consistency Models（一致性模型, 2023）

**论文**：Song et al., "Consistency Models", ICML 2023

**核心思路**：解决扩散模型采样慢的问题，训练模型使得 ODE 轨迹上任意点都能**一步**映射到轨迹起点 $x_0$。

**核心性质（自一致性）**：

$$f_\theta(x_t, t) = f_\theta(x_{t'}, t'), \quad \forall t, t' \in [\epsilon, T]$$

即对于同一条 PF-ODE 轨迹上的所有点，模型的输出应当相同（都等于 $x_0$）。

**训练方式**：

- **蒸馏（Distillation）**：从预训练的扩散模型中蒸馏
- **直接训练（Training from scratch）**：不依赖预训练模型

**核心优势**：支持 1 步或少步（2-4 步）高质量生成。

### 3.7 Flow Matching（流匹配, 2023）

**论文**：Lipman et al., "Flow Matching for Generative Modeling", ICLR 2023

**核心思路**：用**最优传输（Optimal Transport）**的视角重新理解扩散过程，训练一个向量场将噪声分布传输到数据分布。

**与扩散模型的关系**：

- 扩散模型的概率流 ODE 可以看作一种 Flow Matching 的特例
- Flow Matching 提供了更直的传输路径，减少了采样步数
- 数学上更简洁，训练更高效

**代表应用**：Stable Diffusion 3 采用了 Flow Matching 框架替代传统扩散。

### 3.8 算法演进总结

```
DDPM (2020)        ──→  奠基：证明扩散模型生成质量可媲美 GAN
    │
    ├─→ DDIM (2021)          ──→  加速：跳步采样，50 步即可
    │
    ├─→ Score SDE (2021)     ──→  统一：连续时间统一框架
    │
    ├─→ Classifier-Free      ──→  可控：无分类器条件引导
    │    Guidance (2022)
    │
    ├─→ Latent Diffusion     ──→  高效：潜空间扩散，实用化
    │    (2022)
    │
    ├─→ Consistency Models   ──→  极速：1 步生成
    │    (2023)
    │
    └─→ Flow Matching        ──→  优化：更优传输路径
         (2023)
```

## 4. 在机器人领域的应用

扩散模型在机器人学中的应用是近年来最令人兴奋的研究方向之一。其核心价值在于：**将机器人的动作规划问题转化为条件生成问题**。

### 4.1 Diffusion Policy（扩散策略）

**论文**：Chi et al., "Diffusion Policy: Visuomotor Policy Learning via Action Diffusion", RSS 2023

**核心思想**：

将机器人策略 $\pi(a|o)$（从观测到动作的映射）建模为一个**条件扩散模型**。不是预测单个动作，而是通过去噪过程**生成整条动作轨迹**。

**架构**：

```
观测 o_t（图像 + 机器人状态）
    │
    ▼
[观测编码器]（如 ResNet / ViT）
    │
    ▼
观测特征 ──→ [条件扩散模型] ←── 噪声动作序列 a_T
                    │
                    ▼ (迭代去噪 K 步)
                    │
            去噪后的动作序列 a_0 = [a_{t}, a_{t+1}, ..., a_{t+H}]
                    │
                    ▼
            执行前几步动作，然后重新规划（receding horizon）
```

**为什么扩散模型适合机器人策略？**

1. **多模态动作分布建模**：面对同一场景（如桌上有杯子），机器人可以从左边抓、从右边抓、用不同手势抓——这些都是合理的动作。传统回归模型（如 MLP）会输出这些模态的平均值（一个无意义的中间动作），而扩散模型能采样出任一合理模态。

2. **时序一致性**：一次生成整条轨迹（而非逐步预测），避免了动作序列中的抖动和不连贯。

3. **训练稳定性**：相比 GAN-based 策略（如 IBC），扩散策略的训练更稳定，无需对抗训练。

**实际效果**：在多个操控基准任务（推T形块、翻杯子等）上，Diffusion Policy 显著超越 BC-MLP、BC-GMM、IBC、LSTM-GMM 等基线方法。

### 4.2 Diffuser（扩散规划器）

**论文**：Janner et al., "Planning with Diffusion for Flexible Behavior Synthesis", ICML 2022

**核心思想**：

将**整条轨迹（状态-动作序列）**作为扩散模型的生成目标，从而将规划问题转化为条件生成问题。

**关键创新**：

- 将 $(s_0, a_0, s_1, a_1, ..., s_T, a_T)$ 拼接为一个 2D 数组，作为"图像"输入 U-Net
- 通过**inpainting**实现条件约束：
  - 起始状态约束：在每步去噪后，将 $s_0$ 替换为当前真实状态
  - 目标约束：将 $s_T$ 替换为目标状态
  - 奖励引导：使用 classifier guidance 引导轨迹朝高回报方向生成

```
生成目标:
┌─────────────────────────────────────────────┐
│  s₀  a₀  s₁  a₁  s₂  a₂  ...  sT  aT    │  ← 完整轨迹
└─────────────────────────────────────────────┘
     ▲                              ▲
     │                              │
   固定为当前状态              可固定为目标状态
   (inpainting)               (inpainting)
```

**优势**：

- 同一个模型可以通过不同的条件方式实现不同任务（目标到达、奖励最大化、避障等）
- 天然支持长时序推理
- 不需要单独的世界模型和策略网络

### 4.3 UMI（Universal Manipulation Interface）中的扩散策略

**核心架构**：

UMI 使用扩散策略作为其动作生成的核心。它的特点是结合了：
- 手持夹爪采集的人类演示数据
- 基于扩散模型的策略网络
- 相对位姿表示（相对于夹爪的动作表示）

通过在潜空间中运行扩散过程，UMI 实现了高效的跨具身（cross-embodiment）迁移。

### 4.4 ScrewMimic / 装配任务中的扩散模型

**应用场景**：精密装配（螺丝拧紧、插销插入等）

**核心思路**：

- 力/力矩信号 + 视觉观测作为条件
- 扩散模型生成 SE(3) 空间中的动作轨迹
- 处理接触丰富（contact-rich）操控中的多模态和不确定性

### 4.5 SE(3)-DiffusionFields

**论文**：Urain et al., "SE(3)-DiffusionFields: Learning Smooth Cost Functions for Joint Grasp and Motion Optimization", ICRA 2023

**核心思路**：

- 在 SE(3) 流形（位置 + 姿态）上定义扩散过程
- 学习抓取位姿的分布
- 扩散模型的能量/分数可以作为**平滑的代价函数**，与轨迹优化器结合

### 4.6 机器人领域应用汇总

| 应用方向 | 代表工作 | 扩散模型角色 | 条件输入 |
|---------|---------|------------|---------|
| 模仿学习/策略 | Diffusion Policy | 动作生成器 | 视觉 + 本体感知 |
| 轨迹规划 | Diffuser | 全轨迹规划器 | 起始/目标状态 + 奖励 |
| 抓取规划 | SE(3)-DiffusionFields | 抓取位姿生成 | 点云 |
| 任务与运动规划 | DiffusionTAMP | 联合任务-运动规划 | 任务描述 + 场景 |
| 跨具身策略 | UMI | 通用操控策略 | 视觉 + 相对位姿 |
| 数据增强 | GenAug, RoboGen | 生成训练数据 | 场景描述 |
| 强化学习 | Diffusion-QL | 策略表示 | 状态 + Q值引导 |
| 人形机器人 | HumanPlus | 全身动作生成 | 人类动作捕捉 |

## 5. 在自动驾驶领域的应用

### 5.1 驾驶场景生成

#### 5.1.1 MagicDrive / MagicDrive3D

**核心思路**：

- 利用扩散模型生成**多视角、3D 一致**的驾驶场景图像/视频
- 条件输入：3D bounding box、BEV 地图、文本描述、相机参数
- 用于训练数据增强，解决罕见场景（corner case）数据不足的问题

**架构**：

```
条件输入:
  - 3D Bounding Boxes ──→ [3D 位置编码]
  - BEV 语义地图     ──→ [BEV 编码器]      ──→ [Latent Diffusion]
  - 文本描述         ──→ [CLIP 编码器]          + 多视角一致性约束
  - 相机内外参       ──→ [相机编码]                    │
                                                       ▼
                                              多视角驾驶场景图像
```

#### 5.1.2 DriveDreamer / DriveDreamer-2

**核心思路**：

- 生成时序一致的驾驶视频
- 第一阶段：学习结构化的交通世界模型
- 第二阶段：用扩散模型生成逼真的驾驶视频
- 支持通过文本/HDMap 控制生成内容（如"前方车辆左转"）

### 5.2 运动规划与轨迹生成

#### 5.2.1 Diffusion Planner

**核心思路**：

将自动驾驶的运动规划建模为条件轨迹生成问题：

```
输入:
  - 高精地图 / BEV 特征
  - 周围车辆历史轨迹
  - 交通规则约束
  - 导航意图（直行/左转/右转）
      │
      ▼
[条件扩散模型]
      │
      ▼
输出: 未来 N 秒的自车规划轨迹
```

**优势**：

- 可以建模**多种合理的驾驶行为**（如换道时机的多模态性）
- 通过 guidance 施加安全约束（如碰撞避免、车道保持）
- 比传统优化方法更好地处理复杂交互场景

#### 5.2.2 CTG++（Controllable Traffic Generation）

**核心思路**：

- 同时生成所有交通参与者的联合轨迹
- 通过条件引导实现可控的交通流生成
- 可以指定特定场景（如"两车追尾"、"行人横穿"）

#### 5.2.3 MotionDiffuser

**论文**：Jiang et al., "MotionDiffuser: Controllable Multi-Agent Motion Prediction Using Diffusion", CVPR 2023

**核心思路**：

- 将多智能体运动预测建模为联合分布生成问题
- 用扩散模型同时预测场景中所有车辆的未来轨迹
- 天然建模交互（如让行、汇流等协调行为）

### 5.3 端到端自动驾驶

#### 5.3.1 GenAD（Generalized Autonomous Driving）

**核心思路**：

将端到端自动驾驶统一到一个生成式框架中：

- **感知**：扩散模型生成 BEV 分割和 3D 检测结果
- **预测**：扩散模型预测其他交通参与者的未来轨迹
- **规划**：扩散模型生成自车规划轨迹

统一的生成式框架使得感知、预测、规划之间可以端到端优化。

#### 5.3.2 DiffusionDrive

**核心思路**：

- 用扩散模型直接从传感器输入生成驾驶动作
- 引入**截断扩散策略（Truncated Diffusion Policy）**：不从纯噪声开始，而是从一组"锚轨迹"开始去噪，大幅减少推理步数
- 在 NAVSIM 基准上达到 SOTA，仅需 2 步去噪

### 5.4 3D 感知与重建

#### 5.4.1 LiDiff / LiDAR 点云生成

**核心思路**：

- 用扩散模型生成或补全 LiDAR 点云
- 条件输入：稀疏点云、相机图像、语义标签
- 应用：LiDAR 数据增强、点云超分辨率、跨传感器域适应

#### 5.4.2 占据预测（Occupancy Prediction）

**核心思路**：

- 用扩散模型预测未来的 3D 占据网格
- 建模未来场景的多模态不确定性
- 为规划提供更全面的环境理解

### 5.5 仿真与世界模型

#### 5.5.1 GAIA-1 / GAIA-2

**核心思路**：

- 构建基于扩散模型的**驾驶世界模型（World Model）**
- 给定历史帧和驾驶动作，预测（生成）未来的驾驶视频
- 可用于：
  - 闭环仿真测试
  - 数据增强
  - 策略评估（无需真实上路）

#### 5.5.2 UniSim

**核心思路**：

- 构建统一的多传感器仿真器
- 用扩散模型同时生成相机图像和 LiDAR 点云
- 支持场景编辑（修改车辆位置、添加行人等）

### 5.6 自动驾驶领域应用汇总

| 应用方向 | 代表工作 | 输入 | 输出 |
|---------|---------|------|------|
| 场景图像生成 | MagicDrive | 3D Box + BEV + 文本 | 多视角驾驶图像 |
| 驾驶视频生成 | DriveDreamer | HDMap + 文本 | 时序一致驾驶视频 |
| 运动规划 | DiffusionDrive | 传感器 + 地图 | 自车规划轨迹 |
| 多智能体预测 | MotionDiffuser | 历史轨迹 + 地图 | 联合未来轨迹 |
| 交通流生成 | CTG++ | 场景描述 + 约束 | 多车联合轨迹 |
| 端到端驾驶 | GenAD | 传感器原始数据 | 感知+预测+规划 |
| LiDAR 生成 | LiDiff | 稀疏点云/图像 | 稠密 LiDAR 点云 |
| 世界模型 | GAIA-2 | 历史帧+动作 | 未来驾驶视频 |
| 占据预测 | OccDiffusion | BEV 特征 | 未来 3D 占据 |
| 仿真 | UniSim | 场景编辑指令 | 多传感器仿真 |

## 6. 总结与展望

### 6.1 扩散模型的核心优势

1. **训练稳定**：简单的去噪目标，不需要对抗训练，不需要复杂的损失函数平衡
2. **分布覆盖全面**：天然避免模式崩溃，能建模复杂的多模态分布
3. **灵活的条件控制**：通过 Classifier-Free Guidance 等技术，轻松实现各种条件生成
4. **理论基础坚实**：与变分推断、分数匹配、随机微分方程等理论体系相连

### 6.2 当前主要挑战

| 挑战 | 现状 | 研究方向 |
|------|------|---------|
| **推理速度** | 仍需多步迭代（2-50 步） | Consistency Models, 蒸馏, Flow Matching |
| **实时性** | 机器人/自驾需毫秒级响应 | 截断扩散, 缓存机制, 硬件加速 |
| **长时序一致性** | 长视频/长轨迹一致性难保证 | 层次化扩散, 时空注意力 |
| **物理合理性** | 生成结果可能违反物理定律 | 物理约束引导, 物理模拟器结合 |
| **安全保障** | 生成式策略的安全性难以形式化验证 | 安全引导, 约束优化, 运行时监控 |

### 6.3 未来趋势

1. **扩散模型 + 大语言模型**：用 LLM 提供高层语义理解和任务分解，扩散模型负责低层动作生成
2. **统一的世界模型**：用扩散模型构建通用的物理世界模拟器，同时服务于机器人和自动驾驶
3. **实时扩散**：通过架构优化和蒸馏技术，实现毫秒级扩散推理
4. **Sim-to-Real 弥合**：用扩散模型生成高保真仿真数据，减少 sim-to-real gap
5. **多模态融合**：在统一的扩散框架中融合视觉、语言、触觉、力觉等多模态信息
