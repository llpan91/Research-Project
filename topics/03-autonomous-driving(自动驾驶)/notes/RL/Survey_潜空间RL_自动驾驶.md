# Survey: 潜空间强化学习在自动驾驶中的应用

> 本报告聚焦于 **潜在空间（Latent Space）** 中进行强化学习的方法体系，系统梳理其在自动驾驶中的研究进展，涵盖世界模型、潜在动力学学习、想象规划等核心技术。

---

## 一、引言

### 1.1 什么是潜空间强化学习？

**潜空间强化学习（Latent-Space RL）** 是一类在学到的低维潜在表示空间中进行策略学习、规划和决策的方法。其核心思想是：

```
原始观测空间 (高维)          潜在空间 (低维、紧凑)
  图像: 256×256×3         →    向量: z ∈ R^d (d << 原始维度)
  点云: N×3               →    结构化潜在状态
  多模态传感器             →    统一潜在表示

在潜在空间中: 学习动力学 → 想象未来 → 优化策略
```

### 1.2 为什么在潜空间做 RL？

| 优势 | 说明 |
|------|------|
| **降维** | 原始观测（图像、点云）维度极高，直接在原始空间学习动力学不可行 |
| **样本效率** | 潜在空间紧凑，学习动力学模型更快、所需数据更少 |
| **快速想象** | 在低维空间 rollout 比在原始像素空间快几个数量级 |
| **泛化** | 好的潜在表示可以过滤噪声、捕获本质结构 |
| **多模态融合** | 不同传感器模态可以融合到统一的潜在空间 |

### 1.3 与自动驾驶的结合动机

自动驾驶面临的挑战恰好匹配潜空间 RL 的优势：

- **高维感知**：多路摄像头、LiDAR、雷达 → 需要降维
- **复杂动力学**：多智能体、交通规则、不确定性 → 需要学习环境模型
- **安全约束**：不能在真实环境中大量探索 → 需要在模型中想象
- **实时性要求**：决策延迟要求 < 100ms → 需要高效推理

## 二、技术基础

### 2.1 潜在空间学习方法

潜空间 RL 的基础是学习高质量的潜在表示。核心方法包括：

#### (1) 变分自编码器（VAE）

```
编码器: q_φ(z|x) — 将观测 x 映射到潜在分布
解码器: p_θ(x|z) — 从潜在变量重建观测
目标:   max ELBO = E[log p(x|z)] - KL[q(z|x) || p(z)]
```

**在自动驾驶中**：将摄像头图像编码为紧凑的潜在向量，作为 RL 策略的输入。

#### (2) 循环状态空间模型（RSSM）

```
确定性路径:  h_t = f_θ(h_{t-1}, z_{t-1}, a_{t-1})   [GRU]
随机先验:    p_θ(z_t | h_t)                            [预测]
随机后验:    q_φ(z_t | h_t, o_t)                       [推断]
观测模型:    p_θ(o_t | h_t, z_t)                       [重建]
奖励模型:    p_θ(r_t | h_t, z_t)                       [预测]
```

**RSSM 的双路径设计**是 Dreamer 系列的核心创新：
- **确定性路径 h_t**：捕获可预测的动力学趋势
- **随机路径 z_t**：建模环境的随机性和不确定性

#### (3) 对比学习

```
目标: 使时序相邻状态的表示相近，不相关状态远离
代表: CURL, SPR, DrQ
```

#### (4) 掩码建模

```
目标: 随机掩码部分输入，从剩余信息重建
代表: Masked World Model, MAE-based
```

### 2.2 潜空间动力学学习

学到潜在表示后，核心是学习潜在空间中的 **转移动力学**：

```
给定: (z_t, a_t)
预测: z_{t+1} = f(z_t, a_t)   — 确定性转移
  或: p(z_{t+1} | z_t, a_t)   — 概率转移
```

| 动力学模型类型 | 说明 | 代表 |
|---------------|------|------|
| 确定性 MLP | z_{t+1} = MLP(z_t, a_t) | 简单但无法建模随机性 |
| 随机模型 | 高斯分布 / 离散分类 | RSSM, Dreamer |
| GRU/LSTM | 循环建模长期依赖 | RSSM 的确定性路径 |
| Transformer | 自注意力捕获长程依赖 | STORM, TransDreamer |
| 扩散模型 | 生成式建模复杂分布 | Diffusion World Model |

### 2.3 想象式规划（Imagination-based Planning）

潜空间 RL 的核心范式 — 在学到的模型中"做梦"：

```
Step 1: 从真实状态编码 z_0 = Enc(o_0)
Step 2: 在潜在空间展开 H 步想象轨迹
         z_0 → a_0 → z_1 → a_1 → ... → z_H
Step 3: 用 reward model 预测每步奖励 r̂_t
Step 4: 计算想象回报，更新策略
```

**关键优势**：
- 一次真实交互 → 数百/数千条想象轨迹
- 想象成本远低于真实交互
- 可以安全地想象危险场景

## 三、核心方法体系

### 3.1 Dreamer 系列（Hafner et al.）

Dreamer 系列是潜空间 RL 最具影响力的工作，奠定了该领域的基础架构：

#### DreamerV1 (2020)

```
核心贡献:
- 首次实现纯想象训练的 Actor-Critic
- RSSM 世界模型 + 潜在空间 policy learning
- 在 DeepMind Control Suite 上超越 model-free 方法

架构: Encoder → RSSM → (Reward Model + Decoder) → Actor-Critic
训练: 真实数据训练世界模型 → 想象轨迹训练策略
```

#### DreamerV2 (2021)

```
改进:
- 离散潜在变量 (32个分类分布，每个32类)
- KL 平衡 (balancing) 技术
- 直接从像素学习 Atari 200M

关键创新: z_t ~ Categorical(32×32) 替代 Gaussian
优势: 离散表示更稳定，避免后验坍塌
```

#### DreamerV3 (2023)

```
改进:
- Symlog 预测 (处理不同尺度的奖励)
- 免超参数调节 (同一组超参跨领域)
- 在 150+ 任务上 SOTA，包括 Minecraft

关键创新:
  - Symlog 编码: symlog(x) = sign(x) * ln(|x| + 1)
  - 自动缩放的 critic 目标
  - 百分位回报归一化
```

### 3.2 面向自动驾驶的潜空间 RL 方法

#### DreamerAD

```
基础: DreamerV3 框架
适配:
  - 自动驾驶观测空间设计 (BEV + 向量)
  - 驾驶特定奖励函数
  - 多步路径点输出
  - 辅助驾驶损失
平台: CARLA
```

#### MILE (Model-Based Imitation Learning for Urban Driving)

```
核心思想:
  - 学习潜在空间世界模型
  - 不用 RL，而用模仿学习训练策略
  - 在潜在空间中做未来预测辅助决策

架构:
  BEV 编码 → 潜在世界模型 → 未来预测
                            → 动作输出

特点:
  - 将 IL 与世界模型结合
  - 潜在空间中的多步预测改善决策
  - NeurIPS 2022
```

#### ISO-Dream (Isolating World Models)

```
核心创新:
  - 将潜在空间分解为 "可控因素" 和 "不可控因素"
  - 可控: 自车运动相关的部分
  - 不可控: 其他车辆、行人等外部动态

分解:
  z_t = [z_t^ctrl, z_t^nonctrl]
  z_{t+1}^ctrl = f(z_t^ctrl, a_t)        ← 仅动作影响
  z_{t+1}^nonctrl = g(z_t^nonctrl)       ← 独立演化

优势: 更准确的可控性建模 → 更好的想象质量
```

#### Think2Drive

```
核心思想:
  - 世界模型 + RL，先"思考"再驾驶
  - 在潜在空间中进行多步前瞻规划
  - 类似 AlphaGo 的"思考"过程用于驾驶

架构:
  感知 → 世界模型 → 多步想象 → 评估+选择 → 执行
```

#### SafeDreamer

```
核心创新:
  - Dreamer + Constrained RL (CMDP)
  - 在想象中同时预测奖励和安全代价
  - 拉格朗日方法约束策略更新

安全约束:
  max  E[Σ r_t]
  s.t. E[Σ c_t] ≤ d    (c_t: 安全代价, d: 约束阈值)

优势: 在世界模型中安全探索，训练过程本身更安全
平台: Safety-Gymnasium, CARLA
```

#### GAIA-1 (Wayve)

```
核心思想:
  - 大规模生成式世界模型
  - 输入: 视频 + 文本 + 动作 → 生成未来驾驶视频
  - 9B 参数的 Transformer 世界模型

特点:
  - 不直接做 RL，而是提供世界模拟能力
  - 可以生成多样化的未来场景
  - 潜在空间建模 + 视频生成

意义: 证明了大规模世界模型在自动驾驶中的可行性
```

#### DriveDreamer 系列

```
DriveDreamer (2023):
  - 基于扩散模型的驾驶世界模型
  - 生成未来驾驶场景视频
  - 可控的场景生成 (控制交通参与者行为)

DriveDreamer-2 (2024):
  - 改进的场景生成质量
  - 更好的长期一致性
  - 支持更复杂的交通场景
```

### 3.3 其他重要潜空间方法

#### TD-MPC / TD-MPC2

```
核心: 时序差分学习 + 模型预测控制 + 潜在空间

架构:
  编码器: z = h(o)
  潜在动力学: z' = d(z, a)
  奖励预测: r̂ = R(z, a)
  值函数: Q(z, a)

规划: 在潜在空间中用 MPPI 做在线规划
      采样多条动作序列 → 模型前向模拟 → 选最优

TD-MPC2 (2024):
  - 单一模型、单一超参跨 104 个任务
  - 80M 参数潜在世界模型
  - 混合专家 (MoE) 架构
```

#### PlaNet (Planning in Latent Space)

```
核心: RSSM + CEM 规划 (无 Actor-Critic)

规划方式:
  - 交叉熵方法 (CEM) 在潜在空间中搜索最优动作序列
  - 每步在线规划，非离线策略

意义: 首次证明纯在潜在空间规划可以解决连续控制
局限: 在线规划计算开销大
```

#### SLAC (Stochastic Latent Actor-Critic)

```
核心: 随机潜在变量模型 + SAC

架构:
  - 序列 VAE 学习潜在状态
  - 在潜在状态上运行 SAC
  - 不做想象 rollout，仅用潜在表示

意义: 证明好的潜在表示本身就能提升 model-free RL
```

#### LEXA (Latent Explorer Achiever)

```
核心: 潜在空间中的目标条件 RL

方法:
  - 在潜在空间中设定目标
  - Explorer 探索新状态
  - Achiever 学习达到指定潜在目标

应用: 可迁移到自动驾驶中的目标导向驾驶
```

### 4.3 自动驾驶特有的设计考量

#### 观测编码

| 方案 | 编码器 | 适用场景 |
|------|--------|----------|
| 前视图像 | CNN / ResNet / ViT | 端到端视觉驾驶 |
| BEV 语义图 | 轻量 CNN | 结构化场景理解 |
| BEV + 向量 | CNN + MLP → 融合 | 多模态 (DreamerAD) |
| 点云 | PointNet / VoxelNet | LiDAR-based |
| 多视角图像 | BEVFormer 等 | 环视感知 |

#### 潜在空间维度选择

```
典型配置:
  - DreamerV3: 32 categorical distributions × 32 classes = 1024 维离散潜在空间
  - 确定性状态 h: 512-4096 维 GRU 隐状态
  - 总状态维度: ~2000-5000 维

自动驾驶中的调优:
  - 城市复杂场景: 更大的潜在空间 (更多分类分布)
  - 简单高速场景: 较小的潜在空间即可
  - 多传感器融合: 需要更大容量
```

#### 想象 Horizon 选择

```
短 Horizon (5-10 步):
  ✓ 模型误差积累少
  ✓ 训练稳定
  ✗ 难以学习长期行为

长 Horizon (15-50 步):
  ✓ 可以学习远期规划
  ✗ 误差积累 (compounding error)
  ✗ 梯度消失/爆炸

自动驾驶中的选择:
  - 通常 H = 15-30 步
  - 对应物理时间 1.5-3.0 秒 (假设 10Hz 决策频率)
  - 需平衡短期反应与长期规划
```

## 五、关键技术挑战与解决方案

### 5.1 复合误差（Compounding Error）

**问题**：世界模型的单步预测误差在多步 rollout 中指数级积累。

```
真实轨迹:   s₀ → s₁ → s₂ → s₃ → ... → s_H
想象轨迹:   s₀ → ŝ₁ → ŝ₂ → ŝ₃ → ... → ŝ_H
误差:        0    ε    2ε    4ε          2^H·ε  (最坏情况)
```

**解决方案**：

| 方案 | 说明 |
|------|------|
| 短 Horizon | 限制想象步数，减少误差积累 |
| 模型集成 (Ensemble) | 多个模型取平均，降低单模型偏差 |
| 定期锚定 | 每隔 K 步用真实观测重新编码 |
| 对抗训练 | 训练模型生成对抗性最坏情况 |
| 不确定性感知 | 检测高不确定性区域，回退到 model-free |

### 5.2 表示崩塌（Representation Collapse）

**问题**：潜在空间退化，所有状态映射到相似的表示。

**解决方案**：

| 方案 | 说明 |
|------|------|
| KL 平衡 | DreamerV2 的先验/后验 KL 平衡技术 |
| 离散潜在变量 | 分类分布比连续高斯更稳定 |
| 重建损失 | 强制潜在状态保留足够信息 |
| 对比损失 | 拉开不同状态的表示距离 |
| 辅助任务 | 预测速度、碰撞等驾驶相关量 |

### 5.3 多智能体建模

**问题**：自动驾驶环境中其他交通参与者的行为难以预测。

```
挑战:
  - 其他车辆/行人的意图未知
  - 交互行为（博弈）难建模
  - 交通参与者数量动态变化

解决思路:
  1. ISO-Dream 式分解: 可控 vs 不可控因素
  2. 注意力机制: 对关键交通参与者加权
  3. 图网络 (GNN): 建模交通参与者间关系
  4. 条件世界模型: 以其他车辆行为为条件
```

### 5.4 长尾场景处理

**问题**：世界模型难以学到罕见但关键的场景（如突然横穿的行人）。

```
解决思路:
  1. 经验优先采样: 对罕见事件赋予更高采样权重
  2. 场景生成: 用世界模型主动生成危险场景
  3. 课程学习: 逐步增加场景复杂度
  4. 数据增强: 在潜在空间中插值/外推生成新场景
```

### 5.5 Sim-to-Real 迁移

**问题**：在仿真中学到的潜在世界模型如何迁移到真实世界？

```
方案                         说明
Domain Randomization         仿真中随机化视觉/物理参数
域自适应 (Domain Adaptation) 对齐仿真与真实的潜在分布
真实数据微调                  用少量真实数据微调世界模型
域不变表示                   学习对域变化鲁棒的潜在表示
仿真保真度提升               缩小 sim-real gap (CARLA→真实)
```

## 七、前沿方向

### 7.1 Transformer 世界模型

```
趋势: 用 Transformer 替代 GRU-based RSSM
代表:
  - STORM: Transformer-based World Model
  - TransDreamer: Transformer + Dreamer
  - GAIA-1: 9B 参数 Transformer 世界模型

优势:
  - 更好的长程依赖建模
  - 可扩展到更大模型
  - 与 Foundation Model 趋势一致

挑战:
  - 计算成本更高
  - 自回归生成的延迟
```

### 7.2 生成式世界模型 (Diffusion/Video)

```
趋势: 用视频生成模型作为世界模拟器
代表:
  - Sora (OpenAI) — 通用视频世界模拟器的概念验证
  - GAIA-1, DriveDreamer — 驾驶场景视频生成
  - GenAD, Vista — 驾驶视频预测

架构演进:
  VAE/RSSM → GAN → Diffusion → Video Foundation Model

意义: 世界模型从 "潜在空间抽象模拟" 走向 "像素级真实模拟"
```

### 7.3 大规模预训练世界模型

```
思路: 像 LLM 一样预训练通用世界模型，再微调到驾驶
路径:
  1. 大规模驾驶视频 → 预训练通用驾驶世界模型
  2. 特定任务/场景 → 微调世界模型 + RL 策略
  3. 多任务、多域统一模型

代表思路:
  - UniSim (Google): 通用交互式世界模拟器
  - Wayve PRISM: 大规模驾驶世界模型
```

### 7.4 层级化潜空间规划

```
思路: 多尺度/多层次的潜在空间
高层: 抽象意图空间 (去哪里、做什么行为)
中层: 轨迹空间 (路径规划)
底层: 控制空间 (具体控制量)

每层对应不同时间尺度:
  高层: 5-10秒  → 行为决策
  中层: 1-3秒   → 轨迹规划
  底层: 0.1秒   → 控制执行
```

### 7.5 安全约束的潜空间 RL

```
趋势: 在世界模型想象中嵌入安全保证
方法:
  1. SafeDreamer: CMDP + 世界模型
  2. 安全值函数: 在潜在空间中学习安全边界
  3. 可达性分析: 在潜在空间中计算安全可达集
  4. Shield: 在想象阶段过滤不安全轨迹
```

### 7.6 LLM + 潜空间世界模型

```
思路: LLM 提供高层常识推理，世界模型提供物理模拟
架构:
  LLM 理解场景 → 提供高层指导/约束
  世界模型 → 在潜在空间中模拟物理演化
  RL 策略 → 在两者指导下学习决策

优势: 结合语言的可解释性与世界模型的精确模拟
```

## 八、总结

### 8.1 核心洞察

> **潜空间 RL 通过 "学习世界的压缩表示 → 在想象中训练策略" 这一范式，为自动驾驶提供了一条兼顾样本效率、安全性和性能的技术路线。**

### 8.2 关键结论

| 结论 | 说明 |
|------|------|
| 潜在表示是核心 | 好的潜在空间决定了世界模型质量和策略上限 |
| 想象训练是关键 | 大幅提升样本效率，允许安全探索 |
| 离散优于连续 | DreamerV2/V3 证明离散潜在变量更稳定 |
| 复合误差是瓶颈 | 长 horizon 想象仍受限于误差积累 |
| 领域适配不可少 | 通用方法需要驾驶特定的观测/奖励/动作设计 |
| 安全是必要条件 | 纯性能优化不够，需嵌入安全约束 |

### 8.3 发展路线图

```
过去 (2019-2021):
  PlaNet → Dreamer → DreamerV2
  证明潜空间 RL 的可行性

现在 (2022-2024):
  DreamerV3, DreamerAD, SafeDreamer, MILE, Think2Drive
  面向自动驾驶的适配与优化

未来 (2025+):
  大规模预训练世界模型 + LLM 融合 + 安全保证
  通用、可靠、可部署的驾驶 agent
```
