## 1. 怎么理解 “在数学上等价于学习每个噪声水平下数据分布的梯度场”

### 先理解”梯度场”是什么

想象数据分布 $p(x)$ 是一片山地地形——数据密集的区域是山峰（高概率），数据稀疏的区域是山谷（低概率）。**梯度场** $\nabla_x \log p(x)$ 就是这片地形上每个点的”上坡方向”——一个向量，告诉你从当前位置走哪个方向、走多远可以到达更高概率（更真实的数据）的位置。

这个梯度场也叫**分数函数（score function）**。

### 为什么”预测噪声”等价于”学习梯度场”

核心等式：

$$s_\theta(x_t, t) = \nabla_{x_t} \log p_t(x_t) \approx -\frac{\varepsilon_\theta(x_t, t)}{\sqrt{1 - \bar{\alpha}_t}}$$

这里的关键推导链是：

1. **前向过程**告诉我们：$x_t = \sqrt{\bar{\alpha}_t}\, x_0 + \sqrt{1 - \bar{\alpha}_t}\, \varepsilon$，即含噪图像 = 缩放后的原图 + 缩放后的噪声
2. 如果网络能准确预测加入的噪声 $\varepsilon$，那本质上它知道了 $x_t$ 中”哪些成分是噪声、哪些是信号”
3. **”指向信号方向”** = **”远离噪声方向”** = $-\varepsilon$（反方向）
4. 除以尺度因子 $\sqrt{1-\bar{\alpha}_t}$ 后，这恰好就是分数函数 $\nabla_{x_t} \log p_t(x_t)$

### 直觉理解

| 网络在做的事情 | 等价的数学含义 |
|-------------|-------------|
| 看到一张加了噪声的图，预测其中的噪声 $\varepsilon$ | 学习 $t$ 时刻数据分布的梯度场（分数函数） |
| 噪声预测越准，去噪方向越精确 | 梯度场越精确，沿梯度场走越能到达高概率区域 |


“每个噪声水平下”的含义：不同的 $t$ 对应不同程度的噪声污染（$t$ 小时几乎是清晰图像，$t$ 大时几乎是纯噪声），网络需要在**每个噪声水平**都能正确指出去噪方向——这就是在整个噪声水平谱上学习了梯度场。

**一句话总结**：训练网络预测噪声，数学上就是在让它学习”从任意噪声程度出发，朝真实数据走的方向和步长”。

## 2. 为何 DDIM / Flow Matching 有效？
> DDIM / Flow Matching 本质上是对同一个梯度场做不同的 ODE 积分路径。

**ODE（Ordinary Differential Equation，常微分方程）** 描述的是一个量如何随时间连续变化。形式为：

$$\frac{dx}{dt} = v(x, t)$$

含义：在任意时刻 $t$、任意位置 $x$，$v(x,t)$ 告诉你 $x$ 下一瞬间的变化方向和速度。给定初始值 $x(T)$，沿着这个”速度场”从 $T$ 积分到 $0$，就能得到终点 $x(0)$。

**关键性质**：ODE 是确定性的——给定同一个起点，轨迹唯一确定。

### 扩散模型的梯度场定义了一个 ODE

文档 §3.2 中，Score SDE（SDE = Stochastic Differential Equation，随机微分方程）框架将扩散模型的逆向过程写成：

$$dx = \underbrace{\left[f(x,t) - g(t)^2 \nabla_x \log p_t(x)\right]}_{\text{速度场 } v(x,t)} dt + g(t)\, d\bar{w}$$

如果去掉随机项 $g(t) d\bar{w}$，就得到一个纯 ODE（叫 **Probability Flow ODE**）：

$$dx = v(x,t)\, dt$$

其中速度场 $v(x,t)$ 完全由**学到的梯度场（分数函数）** $\nabla_x \log p_t(x)$ 决定。

### 为何 DDIM 有效

**DDIM（Denoising Diffusion Implicit Models）** 是 Song et al. (2020) 提出的加速采样方法，它恰好就是对这个 Probability Flow ODE 的**离散化数值求解**：

- **DDPM（Denoising Diffusion Probabilistic Models，去噪扩散概率模型）**：Ho et al. (2020) 提出的经典扩散模型，每一步都加随机噪声（对应完整的 SDE，随机过程，必须小步走）
- DDIM（$\sigma_t=0$）：去掉随机项，变成确定性 ODE → 可以**跳步**（选择更大的步长），因为 ODE 的轨迹是光滑的

打个比方：DDPM 像是在暴风雨中走山路（每步都有随机风力干扰），必须小心翼翼一步步走；DDIM 像是在无风天走同一条路，因为路径可预测，所以可以大步跨越。

### 为何 Flow Matching 有效

**Flow Matching（流匹配）** 也在学习一个速度场 $v_\theta(x,t)$，但它选择了**更直的传输路径**：

| 方法 | 路径形状 | 效率 |
|------|---------|------|
| DDPM (SDE) | 随机路径（每步加随机噪声） | 必须小步走，~1000 步 |
| DDPM/DDIM 的 ODE | 弯曲路径（由高斯加噪过程决定） | 路径弯曲 → 数值积分需要更多步，~50 步 |
| Flow Matching | 近似直线路径（最优传输） | 路径更直 → 更少步数就能到达终点，~10-20 步 |
| | | |

**但本质是同一回事**：两者都是在学习一个向量场，将噪声分布传输到数据分布。区别只在于：
- 向量场对应的传输路径不同（弯曲 vs 直线）
- 数值积分（ODE 求解）所需的步数因此不同

### 统一理解

```
噪声分布 N(0,I)  ──────→  数据分布 p(x₀)
        │                         │
        │    沿梯度场/速度场积分     │
        │    (ODE 数值求解)         │
        │                         │
        └── DDIM: 弯曲路径, ~50步 ──┘
        └── Flow Matching: 直线路径, ~10-20步 ──┘
        └── DDPM (SDE): 随机路径, ~1000步 ──┘
```

**一句话总结**：扩散模型学到的梯度场定义了一个从噪声到数据的向量场，DDIM 和 Flow Matching 只是用不同的路径和步长在这个向量场上做数值积分——殊途同归。

## 3. 扩散模型的理论基础：SDE + Score Matching

### 分数匹配（Score Matching）

**分数（Score）** 指的是对数概率密度的梯度：$\nabla_x \log p(x)$。

**问题**：我们想知道真实数据分布 $p(x)$ 的分数函数，但 $p(x)$ 本身未知（只有样本数据）。

**分数匹配的核心思想**：不需要知道 $p(x)$ 的具体形式，只需要训练一个神经网络 $s_\theta(x)$ 去逼近 $\nabla_x \log p(x)$。

**怎么训练？** 直接最小化：

$$\mathbb{E}_{p(x)}\left[\|s_\theta(x) - \nabla_x \log p(x)\|^2\right]$$

这个目标看似需要知道真实分数 $\nabla_x \log p(x)$（未知的），但 Vincent (2011) 证明了一个巧妙等价：**去噪分数匹配（Denoising Score Matching）**——给数据加噪声 $\tilde{x} = x + \sigma\varepsilon$，然后训练网络预测加入的噪声方向，就等价于学习了加噪后分布 $p_\sigma(\tilde{x})$ 的分数函数。

**这就是 DDPM 损失的理论根源**：预测噪声 $\varepsilon$ 的 MSE（Mean Squared Error，均方误差）损失 = 去噪分数匹配。

### SDE（随机微分方程）

**SDE（Stochastic Differential Equation）** 是描述含随机性的连续时间过程的数学工具。形式为：

$$dx = f(x,t)\,dt + g(t)\,dw$$

| 组成部分 | 含义 | 类比 |
|---------|------|------|
| $f(x,t)\,dt$ | 确定性漂移项 | 河水的主流方向 |
| $g(t)\,dw$ | 随机扩散项（$dw$ 是布朗运动） | 水分子的随机热运动 |


**在扩散模型中的作用**：

Song et al. (2021) 将 DDPM 的离散加噪/去噪过程推广为连续时间的 SDE：

- **前向 SDE**（数据 → 噪声）：$dx = f(x,t)\,dt + g(t)\,dw$
  - 将干净数据逐渐变成高斯噪声
- **逆向 SDE**（噪声 → 数据）：$dx = [f(x,t) - g(t)^2 \nabla_x \log p_t(x)]\,dt + g(t)\,d\bar{w}$
  - 逆向唯一需要学习的就是分数函数 $\nabla_x \log p_t(x)$

### SDE 与Score Matching如何结合

```
分数匹配：提供了”学什么”的答案
  → 学习每个噪声水平下的分数函数 ∇_x log p_t(x)

SDE：提供了”怎么用”的框架
  → 将学到的分数函数代入逆向 SDE，从噪声生成数据
```

| 概念 | 角色 | 回答的问题 |
|------|------|----------|
| **分数匹配** | 训练目标 | 网络该学什么？→ 学每个噪声水平的梯度场 |
| **SDE** | 统一框架 | 如何用学到的梯度场生成数据？→ 解逆向 SDE/ODE |


**一句话总结**：分数匹配告诉我们”预测噪声 = 学习梯度场”，SDE 告诉我们”有了梯度场就能逆转扩散过程，从噪声回到数据”。两者合在一起，构成了扩散模型的完整理论闭环。

## 4. Inpainting 是什么意思

### 原始含义（图像领域）

**Inpainting（图像修复/补全）** 是指：**固定图像的某些区域不变，让模型只生成/填补剩余区域**。

例如：一张照片中间有个洞（被遮挡或损坏），inpainting 就是让模型根据周围的像素信息，把洞”补”上合理的内容。

### 在扩散模型中的实现

在扩散模型的去噪采样过程中，inpainting 非常自然：

- **已知区域**：每步去噪后，将已知像素**强制替换回真实值**
- **未知区域**：由模型正常去噪生成

这样模型在生成未知区域时，会自然地与已知区域保持视觉一致性。

### 在 Diffuser（轨迹规划）中的应用

文档 §7.4 中，Diffuser 将整条轨迹 $(s_0, a_0, s_1, a_1, ..., s_T, a_T)$ 当作”图像”来生成。此时 **inpainting 被用来施加约束条件**：

```
生成目标（整条轨迹）：
┌─────────────────────────────────────────────┐
│  s₀  a₀  s₁  a₁  s₂  a₂  ...  sT  aT    │
└─────────────────────────────────────────────┘
     ▲                              ▲
     │                              │
   inpainting: 固定为当前状态    inpainting: 固定为目标状态
   （已知，每步去噪后强制替换）   （已知，每步去噪后强制替换）
```

| 图像 inpainting | 轨迹 inpainting |
|----------------|----------------|
| 固定已知像素，生成缺失区域 | 固定起始/目标状态，生成中间轨迹 |
| 保证生成区域与已知区域视觉连贯 | 保证生成轨迹从起点出发、到达终点 |
| | |

**一句话总结**：Inpainting 就是”部分固定、部分生成”——在 Diffuser 中，它是一种优雅的方式来告诉扩散模型”轨迹必须从这里开始、到那里结束，中间怎么走你来生成”。

## 5. 扩散模型天然多模态的底层原因

> 为什么分数函数”指向数据密度高的所有方向，多个峰都被建模”？

### 核心区别：场 vs 点

分数函数 $\nabla_x \log p(x)$ 是定义在**整个空间上的向量场**——在空间中的每个位置都有一个独立的向量值。它不是一个单一输出，而是一张覆盖全空间的”方向地图”。

假设数据分布有两个峰（如左抓和右抓两种动作）：

```
p(x)
 ▲
 │   ╱╲          ╱╲
 │  ╱  ╲        ╱  ╲
 │ ╱    ╲      ╱    ╲
 │╱      ╲____╱      ╲___
 └──────────────────────→ x
   峰A(左抓)    峰B(右抓)

分数函数 ∇log p(x) 的方向：
   ← ← → → →  ← ← → → →
       ↑ 指向峰A    ↑ 指向峰B
```

在峰 A 附近的点，梯度指向峰 A；在峰 B 附近的点，梯度指向峰 B。**两个峰的信息同时编码在这张向量场里**，不存在任何”选择”或”平均”。

### 对比：高斯策略为什么失败

高斯策略 $\pi(a|s) = \mathcal{N}(\mu_\theta(s), \sigma^2)$（其中 $\pi(a|s)$ 是在状态 $s$ 下选择动作 $a$ 的策略分布，$\mu_\theta$ 是网络输出的动作均值）只输出**一个均值点** $\mu$：

| 策略类型 | 面对多模态（左抓 $x=-1$，右抓 $x=+1$） | 结果 |
|---------|--------------------------------------|------|
| 高斯策略 | $\mu = (-1+1)/2 = 0$（被 MSE 拉向平均值） | 从中间抓 → 失败 |
| 扩散模型策略 | 不同初始噪声 → 有的轨迹流向左抓，有的流向右抓 | 每次都合理 |


**根本区别**：高斯策略用**一个点**概括分布，扩散模型用**一个场**描述分布。点必须折中，场可以同时保留所有峰。

### 去噪过程中多模态的实现机制

关键在于**初始噪声的随机性**决定最终落入哪个峰：

```
t=T (纯噪声)     t=T/2 (中间)      t=0 (去噪完成)

  ·A                ·A                 ·A → 峰A(左抓)
     ·B               ·B                 ·B → 峰B(右抓)
  ·C                ·C                 ·C → 峰A(左抓)

三个不同的初始噪声样本，在分数场的引导下，
沿着不同的轨迹流向不同的峰
```

- **$t$ 大时**（高噪声）：分布被噪声平滑成一个大高斯，分数场提供粗粒度的全局方向
- **$t$ 小时**（低噪声）：分布的多峰结构显现，分数场在不同区域指向不同的峰，轨迹”分流”

### 数学上的严格理由

对于混合分布 $p(x) = \sum_k w_k \, p_k(x)$（$w_k$ 为各分量权重，$p_k(x)$ 为各分量分布），其分数函数为：

$$\nabla_x \log p(x) = \frac{\sum_k w_k \, p_k(x) \, \nabla_x \log p_k(x)}{\sum_k w_k \, p_k(x)}$$

这是各分量分数函数的**加权组合**，权重是各分量在当前位置 $x$ 处的**后验概率**。在峰 A 附近，$p_A(x)$ 主导，分数指向峰 A；在峰 B 附近，$p_B(x)$ 主导，分数指向峰 B。神经网络 $\varepsilon_\theta$ 通过 MSE 训练逼近这个场，自然就学会了所有峰的信息。

**一句话总结**：高斯策略用一个点表示策略 → 多峰必须取平均 → 崩溃。扩散模型学的是整个空间的梯度场 → 不同区域自然指向不同的峰 → 多模态被完整保留。初始噪声的随机性决定最终落入哪个峰。
