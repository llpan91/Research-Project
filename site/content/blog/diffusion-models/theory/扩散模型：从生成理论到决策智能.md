## 1. 统一视角：扩散模型的数学本质

### 1.1 一个贯穿全文的核心等式

在深入细节之前，先建立本文最重要的等式：

$$\boxed{s_\theta(x_t, t) = \nabla_{x_t} \log p_t(x_t) \approx -\frac{\varepsilon_\theta(x_t, t)}{\sqrt{1 - \bar{\alpha}_t}}} \tag{1}$$

| 符号 | 含义 |
|------|------|
| $s_\theta(x_t, t)$ | 参数为 $\theta$ 的分数函数网络，近似真实分布梯度场 |
| $\nabla_{x_t} \log p_t(x_t)$ | $t$ 时刻数据分布对数密度关于 $x_t$ 的梯度（真实分数函数） |
| $\varepsilon_\theta(x_t, t)$ | 神经网络对加入噪声的预测值 |
| $\bar{\alpha}_t$ | 前 $t$ 步噪声调度的累积乘积 $\prod_{s=1}^t (1-\beta_s)$ |
| $\sqrt{1 - \bar{\alpha}_t}$ | $t$ 时刻噪声分量的标准差 |

这个等式说明：**训练去噪网络 $\varepsilon_\theta$ 来预测噪声，在数学上等价于学习每个噪声水平下数据分布的梯度场（分数函数）$s_\theta$**。

这个等式是理解以下所有内容的钥匙：

| 现象 | 本质解释（通过上述等式） |
|------|----------------------|
| 为何 MSE 损失能生成高质量图像 | MSE 训练 $\varepsilon_\theta$ ≡ 学习每个 $t$ 下的分数函数 |
| 为何扩散模型天然多模态 | 分数函数指向数据密度高的所有方向，多个峰都被建模 |
| 为何奖励信号可以注入去噪过程 | 奖励梯度 $\nabla J$ 与分数函数 $s_\theta$ 是同类信号，可以叠加 |
| 为何 DDIM / Flow Matching 有效 | 它们本质上是对同一个梯度场做不同的 ODE 积分路径 |

### 1.2 生成模型家族中的位置

```
                        深度生成模型
                            │
            ┌───────────────┼───────────────────┐
            │               │                   │
        显式密度          隐式密度             其他
            │               │                   │
    ┌───────┼───────┐       │            ┌──────┼──────┐
    │       │       │       │            │      │      │
   VAE    Flow   扩散模型   GAN       Autoregressive    EBM
                   ▲
                   │
          与 Score-based Models
          在 SDE（Stochastic Differential Equation，随机微分方程）框架下完全等价
```

**扩散模型 = 层次化 VAE + 固定编码器 + 多步隐变量**

- VAE：可学习编码器，一个隐变量 $z$，一步解码
- 扩散模型：固定加噪过程（无需学习的"编码器"），$T$ 个中间隐变量 $x_1,...,x_T$，多步迭代解码

**关键差异汇总（与主流方法对比）**：

| 对比维度 | GAN | VAE | Normalizing Flow | **扩散模型** |
|---------|-----|-----|-----------------|------------|
| 训练稳定性 | 不稳定（对抗博弈） | 稳定 | 稳定 | **最稳定（纯 MSE）** |
| 生成多样性 | 模式崩溃 | 受限 | 中等 | **最高** |
| 生成速度 | 极快（一步） | 一步 | 一步 | **较慢（多步）** |
| 理论基础 | 博弈论 | 变分推断 | 最优传输 | **SDE + 分数匹配** |
| 条件控制 | 需额外设计 | 有限 | 有限 | **天然支持多种引导** |

## 2. 前向过程：构建训练数据

### 2.1 马尔可夫加噪链

前向过程是**固定的、无参数的**——它不需要学习，只是一个用来构造训练样本的工具。

给定真实数据 $x_0$，每步添加少量高斯噪声：

$$q(x_t \mid x_{t-1}) = \mathcal{N}\!\left(x_t;\, \sqrt{1 - \beta_t}\, x_{t-1},\; \beta_t \mathbf{I}\right) \tag{2}$$

| 符号 | 含义 |
|------|------|
| $q(x_t \mid x_{t-1})$ | 前向加噪过程的单步转移概率 |
| $\mathcal{N}(\cdot;\, \mu,\, \Sigma)$ | 均值为 $\mu$、协方差为 $\Sigma$ 的高斯分布 |
| $\sqrt{1 - \beta_t}$ | 单步信号保留系数 |
| $\beta_t$ | $t$ 时刻预设的噪声强度（噪声调度表中的值） |
| $\mathbf{I}$ | 单位矩阵（各维度独立同方差） |

其中 $\beta_t$ 是预设的**噪声调度表**，控制每步加入多少噪声。

### 2.2 闭式采样：训练效率的关键技巧

无需逐步执行 $T$ 步加噪。定义累积信噪比：

$$\alpha_t = 1 - \beta_t, \qquad \bar{\alpha}_t = \prod_{s=1}^{t} \alpha_s \tag{3}$$

| 符号 | 含义 |
|------|------|
| $\alpha_t$ | $t$ 时刻的单步信号保留率 |
| $\beta_t$ | $t$ 时刻的噪声强度 |
| $\bar{\alpha}_t$ | 前 $t$ 步的累积信号保留率 |
| $\prod_{s=1}^{t}$ | 对时间步 $1$ 到 $t$ 的连乘 |

则可以**从 $x_0$ 一步直接采样任意时刻的噪声图像**：

$$x_t = \sqrt{\bar{\alpha}_t}\, x_0 + \sqrt{1 - \bar{\alpha}_t}\, \varepsilon, \qquad \varepsilon \sim \mathcal{N}(0, \mathbf{I}) \tag{4}$$

| 符号 | 含义 |
|------|------|
| $x_t$ | $t$ 时刻的含噪图像 |
| $x_0$ | 原始真实数据 |
| $\sqrt{\bar{\alpha}_t}$ | 信号系数（控制原始信号的保留程度） |
| $\sqrt{1 - \bar{\alpha}_t}$ | 噪声系数（控制噪声分量的强度） |
| $\varepsilon \sim \mathcal{N}(0, \mathbf{I})$ | 从标准高斯分布采样的随机噪声 |

> **这是整个训练框架高效的根本原因**：无论 $T=1000$，每次训练迭代只需一步计算，复杂度 $O(1)$。

### 2.3 噪声调度表：影响训练质量的隐性超参数

| 调度方式 | 公式 | 特点 |
|---------|------|------|
| **Linear Schedule**（Ho 2020） | $\beta_t$ 在 $[10^{-4}, 0.02]$ 线性插值 | 简单，但高分辨率时 $x_T$ 可能不够纯 |
| **Cosine Schedule**（Nichol 2021） | $\bar{\alpha}_t = \cos^2\!\left(\frac{t/T + s}{1 + s} \cdot \frac{\pi}{2}\right)$ | 两端缓慢、中间快速，过渡更平滑 |

Cosine Schedule 的改进逻辑：信噪比不应在 $t=0$ 时骤降，应有充分的"低噪声区域"让模型学习细粒度细节。

### 2.4 训练数据预处理

| 步骤 | 操作 | 理由 |
|------|------|------|
| 像素归一化 | `[0,255] → [-1, 1]` | 与高斯噪声均值对齐 |
| 分辨率统一 | Resize / Center Crop | 保证批次形状一致 |
| 数据增强 | 水平翻转（可选） | 轻量增强，不改变语义分布 |
| 加噪 | 按闭式公式在线生成 | 不预存，每次迭代动态生成 |

## 3. 逆向过程：学习分布的梯度场

前向过程（§2）将数据逐步变成纯高斯噪声。如果我们能**学会逆转这个过程**——从噪声一步步恢复出数据——就得到了一个生成模型。本节先解释逆向过程的完整逻辑链，再进入具体的参数化和网络设计。

### 逆向过程为什么需要"学习"？

从贝叶斯定理出发，真实的单步逆向条件分布为：

$$q(x_{t-1} \mid x_t) = \frac{q(x_t \mid x_{t-1})\, q(x_{t-1})}{q(x_t)}$$

| 符号 | 含义 |
|------|------|
| $q(x_{t-1} \mid x_t)$ | 真实的逆向单步条件分布（我们想要的） |
| $q(x_t \mid x_{t-1})$ | 前向过程的单步转移概率（已知，式 $(2)$） |
| $q(x_{t-1})$ | $t{-}1$ 时刻的边缘分布（对整个数据集的积分，不可解析） |
| $q(x_t)$ | $t$ 时刻的边缘分布（同样不可解析） |

其中 $q(x_t \mid x_{t-1})$ 是前向过程，已知（式 $(2)$）。但 $q(x_{t-1})$ 和 $q(x_t)$ 是各时刻对**整个数据集**积分得到的边缘分布，无法解析计算。

这就是核心困难：**直接计算逆向分布需要知道我们本来就想学习的数据分布本身**。

### 关键突破：前向后验的解析形式

虽然 $q(x_{t-1} \mid x_t)$ 不可直接计算，但如果**同时已知原始数据 $x_0$**，前向后验 $q(x_{t-1} \mid x_t, x_0)$ 是一个**解析高斯分布**：

$$q(x_{t-1} \mid x_t, x_0) = \mathcal{N}\!\left(x_{t-1};\; \tilde{\mu}_t(x_t, x_0),\; \tilde{\beta}_t \mathbf{I}\right) \tag{5}$$

| 符号 | 含义 |
|------|------|
| $q(x_{t-1} \mid x_t, x_0)$ | 已知 $x_0$ 和 $x_t$ 时的前向后验分布（可解析） |
| $\tilde{\mu}_t(x_t, x_0)$ | 前向后验的均值（关于 $x_t$ 和 $x_0$ 的线性组合） |
| $\tilde{\beta}_t$ | 前向后验的方差（仅由噪声调度决定，不含可学习参数） |

其中均值和方差的具体形式为：

$$\tilde{\mu}_t = \frac{\sqrt{\bar{\alpha}_{t-1}}\, \beta_t}{1 - \bar{\alpha}_t}\, x_0 \;+\; \frac{\sqrt{\alpha_t}\,(1 - \bar{\alpha}_{t-1})}{1 - \bar{\alpha}_t}\, x_t \tag{6}$$

$$\tilde{\beta}_t = \frac{1 - \bar{\alpha}_{t-1}}{1 - \bar{\alpha}_t}\, \beta_t \tag{7}$$

| 符号 | 含义 |
|------|------|
| $\tilde{\mu}_t$ | 前向后验均值（$x_0$ 和 $x_t$ 的加权组合） |
| $\bar{\alpha}_{t-1}$ | 前 $t{-}1$ 步的累积信号保留率 |
| $\alpha_t = 1-\beta_t$ | $t$ 时刻的单步信号保留率 |
| $\beta_t$ | $t$ 时刻的噪声强度 |
| $\tilde{\beta}_t$ | 前向后验方差（完全由噪声调度表决定的常量） |

> **直觉**：如果你**已经知道原始图片 $x_0$ 长什么样**，那么从 $x_t$ 退一步到 $x_{t-1}$ 是完全确定的高斯采样——不需要任何学习。

问题在于推理时我们**没有 $x_0$**——$x_0$ 正是我们要生成的目标。

**解决方案**：用神经网络从 $x_t$ **估计** $x_0$（或等价地，估计加入的噪声 $\varepsilon$），然后代入上述解析公式完成一步逆向。这就是为什么逆向过程需要"学习"——我们学的不是逆向分布本身的形式（它是高斯），而是**其中依赖 $x_0$ 的那部分信息**。

### 从噪声预测到梯度场：分数函数的视角

回忆闭式公式（式 $(4)$）：$x_t = \sqrt{\bar{\alpha}_t}\, x_0 + \sqrt{1 - \bar{\alpha}_t}\, \varepsilon$。从中解出 $x_0$：

$$x_0 = \frac{x_t - \sqrt{1 - \bar{\alpha}_t}\, \varepsilon}{\sqrt{\bar{\alpha}_t}}$$

如果网络能预测 $\varepsilon_\theta(x_t, t) \approx \varepsilon$，就能估计 $\hat{x}_0$，进而代入 $\tilde{\mu}_t$ 完成逆向（这就是式 $(13)$ 的来源）。

更深层地，可以证明在 $t$ 时刻含噪分布 $p_t(x_t)$ 中，**数据密度的梯度方向**（即分数函数 $\nabla_{x_t} \log p_t(x_t)$）与噪声方向的关系为：

$$\nabla_{x_t} \log p_t(x_t) = -\frac{\varepsilon}{\sqrt{1 - \bar{\alpha}_t}} \tag{8}$$

| 符号 | 含义 |
|------|------|
| $\nabla_{x_t} \log p_t(x_t)$ | 分数函数：$t$ 时刻数据对数密度关于 $x_t$ 的梯度 |
| $\varepsilon$ | 从 $x_0$ 到 $x_t$ 实际加入的高斯噪声 |
| $\sqrt{1 - \bar{\alpha}_t}$ | 噪声分量的标准差（尺度因子） |

这就是式 $(1)$ 的物理含义：**噪声 $\varepsilon$ 指向远离数据的方向，分数函数指向靠近数据的方向，两者仅差一个符号和尺度因子**。因此：

- 训练网络预测噪声 $\varepsilon_\theta \approx \varepsilon$，**等价于**学习分数函数 $s_\theta \approx \nabla_{x_t} \log p_t(x_t)$
- 逆向去噪的每一步本质上是**沿着数据分布的梯度场方向移动**——从低密度区域（噪声）流向高密度区域（真实数据）

### 逆向过程的完整逻辑链

将上述分析串联，完整描述从纯噪声 $x_T$ 生成数据 $x_0$ 的每一步：

**起点**：从标准高斯分布采样 $x_T \sim \mathcal{N}(0, \mathbf{I})$，此时信号完全被噪声淹没。

**循环**：对 $t = T, T{-}1, \ldots, 1$，依次执行以下五步：

**第一步：噪声预测。** 将当前含噪样本 $x_t$ 和时间步 $t$ 输入神经网络，得到对噪声分量的估计：

$$\hat{\varepsilon} = \varepsilon_\theta(x_t, t) \approx \varepsilon$$

网络的任务是"从 $x_t$ 中辨别出哪些成分是噪声"。

**第二步：等价的分数函数估计。** 由式 $(8)$，噪声预测可直接换算为分数函数估计：

$$s_\theta(x_t, t) = -\frac{\varepsilon_\theta(x_t, t)}{\sqrt{1 - \bar{\alpha}_t}}$$

分数函数 $s_\theta$ 指向数据密度增长最快的方向——即"从当前位置出发，数据最可能在哪个方向"。这正是式 $(1)$ 的核心等式。

**第三步：估计原始数据 $\hat{x}_0$。** 由闭式公式（式 $(4)$）的逆运算，用噪声预测反推出对原始数据的估计：

$$\hat{x}_0 = \frac{x_t - \sqrt{1 - \bar{\alpha}_t}\, \varepsilon_\theta(x_t, t)}{\sqrt{\bar{\alpha}_t}}$$

这是一个"一步到位"的粗略估计——如果网络完美预测了噪声，$\hat{x}_0$ 就精确等于真实 $x_0$。实际中 $\hat{x}_0$ 存在误差，所以需要多步逐渐修正而非一步生成。

**第四步：计算逆向均值。** 将 $\hat{x}_0$ 代入前向后验均值公式（式 $(6)$），得到网络参数化的逆向均值：

$$\mu_\theta(x_t, t) = \frac{\sqrt{\bar{\alpha}_{t-1}}\, \beta_t}{1 - \bar{\alpha}_t}\, \hat{x}_0 \;+\; \frac{\sqrt{\alpha_t}\,(1 - \bar{\alpha}_{t-1})}{1 - \bar{\alpha}_t}\, x_t$$

等价地，将 $\hat{x}_0$ 的表达式展开，可以得到完全用 $\varepsilon_\theta$ 表示的形式（即式 $(13)$）：

$$\mu_\theta(x_t, t) = \frac{1}{\sqrt{\alpha_t}}\left(x_t - \frac{\beta_t}{\sqrt{1 - \bar{\alpha}_t}}\, \varepsilon_\theta(x_t, t)\right)$$

直觉：逆向均值 = 当前 $x_t$ **减去**网络预测的噪声分量（经适当缩放），即"从含噪信号中扣除噪声"。

**第五步：采样 $x_{t-1}$。** 从逆向分布中采样：

$$x_{t-1} \sim \mathcal{N}\!\left(\mu_\theta(x_t, t),\; \sigma_t^2 \mathbf{I}\right)$$

其中 $\sigma_t^2$ 通常取前向后验方差 $\tilde{\beta}_t$（式 $(7)$）或其变体。当 $t=1$ 时直接输出 $\mu_\theta$ 作为 $x_0$（不再加噪声）。

**终点**：经过 $T$ 步迭代后得到 $x_0$，即生成的数据样本。

> **核心概念厘清**：逆向过程不是一个"解密"或"去滤波"的工程操作——它是在一个由分数函数定义的**向量场中的梯度流动过程**，每一步都沿着数据密度增长的方向移动一小步。理解这一点，才能理解为什么同样的去噪框架可以用于图像生成（§6）、轨迹规划（§7）和策略优化（§8）：它们共享的正是"梯度场 + 外部信号叠加"这一统一机制（§6.1），区别仅在于叠加的外部信号是文本语义、奖励函数还是 Q 值。

下面进入逆向过程的具体参数化和网络设计。

### 3.1 逆向过程的参数化

逆向过程是有参数的神经网络来学习的部分：

$$p_\theta(x_{t-1} \mid x_t) = \mathcal{N}(x_{t-1};\; \mu_\theta(x_t, t),\; \Sigma_\theta(x_t, t)) \tag{9}$$

| 符号 | 含义 |
|------|------|
| $p_\theta(x_{t-1} \mid x_t)$ | 参数为 $\theta$ 的逆向（去噪）单步条件概率 |
| $\mu_\theta(x_t, t)$ | 网络预测的逆向均值 |
| $\Sigma_\theta(x_t, t)$ | 网络预测的（或固定的）逆向协方差矩阵 |

实践中网络学习以下三种等价目标之一（可互推）：

$$\varepsilon = \frac{x_t - \sqrt{\bar{\alpha}_t}\,\hat{x}_0}{\sqrt{1 - \bar{\alpha}_t}} \qquad \Longleftrightarrow \qquad \nabla_{x_t} \log p_t(x_t) = -\frac{\varepsilon_\theta}{\sqrt{1 - \bar{\alpha}_t}} \tag{10}$$

| 符号 | 含义 |
|------|------|
| $\varepsilon$ | 真实加入的高斯噪声 |
| $\hat{x}_0$ | 网络对原始数据 $x_0$ 的预测 |
| $\nabla_{x_t} \log p_t(x_t)$ | $t$ 时刻的真实分数函数 |
| $\varepsilon_\theta$ | 噪声预测网络的输出（简写，完整为 $\varepsilon_\theta(x_t, t)$） |
| $\sqrt{1 - \bar{\alpha}_t}$ | 噪声系数，用于在噪声预测与分数函数之间换算 |

| 预测目标 | 符号 | 适用场景 |
|---------|------|---------|
| **噪声（ε-prediction）** | $\hat{\varepsilon} = \varepsilon_\theta(x_t, t)$ | 通用，DDPM 默认 |
| **原图（x₀-prediction）** | $\hat{x}_0 = f_\theta(x_t, t)$ | 部分模型、少步采样 |
| **速度（v-prediction）** | $v = \sqrt{\bar{\alpha}_t}\,\varepsilon - \sqrt{1-\bar{\alpha}_t}\,x_0$ | 少步蒸馏、高分辨率 |
| **分数（score-prediction）** | $s_\theta = \nabla_{x_t} \log p_t(x_t)$ | Score SDE 框架 |

### 3.2 Score SDE：连续时间统一框架

Song et al. (2021) 将离散 DDPM 推广为连续时间的随机微分方程：

- **前向 SDE**：$dx = f(x, t) \, dt + g(t) \, dw$  （数据 → 噪声）
- **逆向 SDE**：$dx = [f(x,t) - g(t)^2 \underbrace{\nabla_x \log p_t(x)}_{\text{分数函数}}] \, dt + g(t) \, d\bar{w}$  （噪声 → 数据）

不同的 SDE 选择对应不同的模型变体：

| SDE 类型 | 对应离散模型 |
|---------|------------|
| VP-SDE (Variance Preserving) | DDPM |
| VE-SDE (Variance Exploding) | NCSN / SMLD |

**关键洞察**：逆向 SDE 中唯一需要学习的就是分数函数 $\nabla_x \log p_t(x)$，这正是 $\varepsilon_\theta$ 在学的东西（差一个尺度因子）。

### 3.3 网络骨干：扩散模型不是新架构，而是新框架

扩散模型本身是一种**训练范式和生成框架**，不限定骨干网络：
> 框架定义"做什么"，骨干网络决定"怎么做"

| 骨干网络 | 应用场景 | 代表工作 |
|---------|---------|---------|
| **U-Net（Time-conditioned）** | 图像生成 | DDPM, Stable Diffusion |
| **Transformer（DiT）** | 大规模图像/视频生成 | DiT, Sora |
| **1D U-Net / MLP** | 机器人轨迹/动作生成 | Diffusion Policy |
| **3D U-Net** | 视频生成 | Video Diffusion Models |
| **Graph Neural Network** | 分子生成 | GeoDiff |

时间步 $t$ 的注入方式：Sinusoidal Embedding → MLP → 注入每个 ResBlock/Attention Block。

## 4. 训练细节：损失、优化与工程实践

### 4.1 标准损失：从 ELBO 到 MSE 的推导路径

**完整推导链**：

$$\underbrace{\max_\theta\; \mathbb{E}[\log p_\theta(x_0)]}_{\text{最大似然}} \;\geq\; \underbrace{\mathrm{ELBO}(\theta)}_{\text{变分下界}} \;\xrightarrow{\text{化简}} \underbrace{L_{\text{simple}} = \mathbb{E}_{t,x_0,\varepsilon}\!\left[\|\varepsilon - \varepsilon_\theta(x_t, t)\|^2\right]}_{\text{噪声预测的 MSE}} \tag{11}$$

| 符号 | 含义 |
|------|------|
| $p_\theta(x_0)$ | 模型对真实数据 $x_0$ 的生成概率（似然） |
| $\mathrm{ELBO}(\theta)$ | 证据下界（Evidence Lower BOund），对数似然的变分下界 |
| $L_{\text{simple}}$ | Ho et al. 提出的简化 MSE 损失（去掉了时间步权重） |
| $\varepsilon$ | 真实噪声 |
| $\varepsilon_\theta(x_t, t)$ | 网络预测噪声 |
| $\mathbb{E}_{t,x_0,\varepsilon}$ | 对时间步 $t$、数据 $x_0$、噪声 $\varepsilon$ 三者联合期望 |

ELBO 中的核心项是 KL 散度：

$$L_{t-1} = \mathbb{E}_q\left[D_{\mathrm{KL}}\!\left(q(x_{t-1} \mid x_t, x_0) \;\|\; p_\theta(x_{t-1} \mid x_t)\right)\right] \tag{12}$$

| 符号 | 含义 |
|------|------|
| $L_{t-1}$ | ELBO 在时间步 $t$ 处的 KL 散度分项 |
| $D_{\mathrm{KL}}(p \Vert q)$ | 从分布 $q$ 到分布 $p$ 的 KL 散度 |
| $q(x_{t-1} \mid x_t, x_0)$ | 已知 $x_0$ 和 $x_t$ 的前向后验分布（解析高斯） |
| $p_\theta(x_{t-1} \mid x_t)$ | 网络建模的逆向分布（需与前向后验对齐） |
| $\mathbb{E}_q$ | 对前向过程 $q$ 的分布取期望 |

由于 $q(x_{t-1} \mid x_t, x_0)$ 是解析可得的高斯分布，其均值为：

$$\tilde{\mu}_t = \frac{1}{\sqrt{\alpha_t}}\left(x_t - \frac{\beta_t}{\sqrt{1 - \bar{\alpha}_t}}\, \varepsilon\right) \tag{13}$$

| 符号 | 含义 |
|------|------|
| $\tilde{\mu}_t$ | 前向后验分布 $q(x_{t-1} \mid x_t, x_0)$ 的解析均值 |
| $\alpha_t = 1-\beta_t$ | 单步信号保留率 |
| $\beta_t$ | $t$ 时刻的噪声强度 |
| $\bar{\alpha}_t$ | 累积信号保留率 |
| $\varepsilon$ | 由闭式公式 $(4)$ 加入的真实噪声 |

让网络预测 $\varepsilon$，KL 退化为预测均值的 L2 距离，即 MSE。Ho et al. 发现去掉时间步权重系数后（$L_{\text{simple}}$），训练反而更稳定——这是一个经验上的重要发现。

### 4.2 损失函数变体

| 损失 | 公式 | 用途 |
|------|------|------|
| $L_{\text{simple}}$（DDPM） | $\mathrm{MSE}(\varepsilon, \hat{\varepsilon})$ | 基础训练目标 |
| $L_{\text{hybrid}}$（IDDPM） | $L_{\text{simple}} + \lambda \cdot L_{\text{vlb}}$ | 同时学习方差，stop-gradient 隔离 |
| $L_{\text{cfg}}$（CFG） | 同 $L_{\text{simple}}$，但 $c$ 以概率 $p$ 替换为 $\varnothing$ | 条件生成训练 |
| $L_{\text{RL+Diffusion}}$（Diffusion-QL） | $L_{\text{simple}} - \alpha \cdot Q_\phi(s, a_0)$ | RL 微调扩散策略 |

> **关系**：$L_{\text{RL+Diffusion}}$ 是将 RL 的 Q 值信号注入扩散损失的直接体现，是本文 §8 的数学基础。

### 4.3 单次训练迭代

```python
for x0 in dataloader:
    x0 = normalize(x0)                          # [-1, 1]
    t  = torch.randint(1, T+1, (B,))            # 随机时间步，均匀采样
    eps = torch.randn_like(x0)                  # 真实噪声

    # 闭式加噪（关键：O(1) 复杂度，不需要迭代）
    sqrt_alpha_bar     = sqrt_alpha_bars[t]
    sqrt_one_minus     = sqrt_one_minus_alpha_bars[t]
    x_t = sqrt_alpha_bar * x0 + sqrt_one_minus * eps

    # 网络预测 + MSE 损失
    eps_pred = model(x_t, t)
    loss = F.mse_loss(eps_pred, eps)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    ema.update(model)                           # EMA 更新，推理时使用 EMA 参数
```

### 4.4 关键优化技巧

| 技术 | 配置 / 说明 | 作用 |
|------|-----------|------|
| **优化器** | AdamW，$\beta_1=0.9, \beta_2=0.999$ | 标准配置 |
| **学习率** | `1e-4 ~ 2e-4`，配合 warmup | 稳定早期训练 |
| **Batch Size** | 256 ~ 2048，可用梯度累积 | 越大越稳定 |
| **EMA** | decay = 0.9999，推理时用 EMA 版本 | **提升 FID 的最重要单项技巧** |
| **梯度裁剪** | `max_norm = 1.0` | 防止梯度爆炸 |

**EMA 更新公式**：

$$\theta_{\text{ema}} \leftarrow \gamma \cdot \theta_{\text{ema}} + (1 - \gamma) \cdot \theta \tag{14}$$

| 符号 | 含义 |
|------|------|
| $\theta_{\text{ema}}$ | 指数移动平均（EMA）维护的平滑参数 |
| $\gamma$ | EMA 衰减系数（通常取 0.9999），越大遗忘越慢 |
| $\theta$ | 当前 SGD/AdamW 更新后的训练参数 |

EMA 是对历史参数的指数平均，相当于对训练轨迹做平滑，显著减少生成高频噪声。

### 4.5 训练与推理的完整流程图

```
┌────────────────────────────────────────────────────────────────┐
│                         训练阶段                                │
│                                                                 │
│  真实图像 x₀ ──→ 随机采样 t ──→ 闭式加噪得 x_t               │
│                                    │                           │
│                          U-Net ε_θ(x_t, t)                     │
│                                    │                           │
│              Loss = MSE(ε̂, ε) ──→ AdamW 反向传播               │
│                                    │                           │
│                          EMA 更新 θ_ema                         │
└────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────┐
│                     推理阶段（DDPM 采样）                        │
│                                                                 │
│  x_T ~ N(0, I)                                                  │
│  for t = T, T-1, ..., 1:                                       │
│      ε̂ = ε_θ_ema(x_t, t)                                      │
│      x_{t-1} = (1/√α_t)(x_t - β_t/√(1-ᾱ_t)·ε̂) + σ_t·z     │
│  return x₀                                                      │
└────────────────────────────────────────────────────────────────┘
```

> **注意**：经典扩散模型优化的是神经网络参数 $\theta$；高斯噪声 $\varepsilon$ 只是固定的"输入信号"，不是可学习参数。

## 5. 采样加速：从慢到快的算法演进

### 5.1 DDPM 的瓶颈

DDPM 需要 $T=1000$ 步推理，每步一次网络前向传播，极慢。解决路径有三：

```
DDPM (2020) ──→ 奠基：证明扩散模型可媲美 GAN
    │
    ├─→ DDIM (2021)           ──→ 加速：跳步 ODE 采样，50 步即可
    │
    ├─→ Score SDE (2021)      ──→ 统一：连续时间框架
    │
    ├─→ Classifier-Free       ──→ 可控：无分类器条件引导
    │    Guidance (2022)
    │
    ├─→ Latent Diffusion      ──→ 高效：潜空间扩散，降低 4-16× 计算量
    │    (2022)
    │
    ├─→ Consistency Models    ──→ 极速：1-4 步生成
    │    (2023)
    │
    └─→ Flow Matching         ──→ 优化：更直的传输路径，更简洁数学
         (2023)
```

### 5.2 DDIM：确定性 ODE 采样

DDIM 将随机马尔可夫链采样替换为**确定性 ODE**：

$$x_{t-1} = \sqrt{\bar{\alpha}_{t-1}} \underbrace{\left( \frac{x_t - \sqrt{1 - \bar{\alpha}_t} \, \epsilon_\theta(x_t, t)}{\sqrt{\bar{\alpha}_t}} \right)}_{\text{预测的 } x_0} + \sqrt{1 - \bar{\alpha}_{t-1} - \sigma_t^2} \cdot \epsilon_\theta(x_t, t) + \sigma_t \epsilon \tag{15}$$

| 符号 | 含义 |
|------|------|
| $x_{t-1}$ | 去噪一步后的输出 |
| $\bar{\alpha}_{t-1}$ | $t-1$ 时刻的累积信号保留率 |
| $\epsilon_\theta(x_t, t)$ | 网络预测的噪声（用于还原 $x_0$ 估计） |
| $\sigma_t$ | 随机性强度，$\sigma_t=0$ 退化为确定性 ODE（DDIM），取特定值退化为 DDPM |
| $\epsilon$ | 额外注入的随机噪声（仅在 $\sigma_t > 0$ 时有效） |

- 当 $\sigma_t = 0$：完全确定性（DDIM），可跳步，无需重新训练
- 当 $\sigma_t$ 取特定值：退化为 DDPM

### 5.3 Consistency Models：一步到位

核心性质（自一致性）：对于同一条 PF-ODE 轨迹上的所有点，输出应相同：

$$f_\theta(x_t, t) = f_\theta(x_{t'}, t'), \quad \forall t, t' \in [\epsilon, T] \tag{16}$$

| 符号 | 含义 |
|------|------|
| $f_\theta(x_t, t)$ | 一致性函数，将任意 $(x_t, t)$ 映射到对 $x_0$ 的估计 |
| $t, t'$ | 同一条 PF-ODE 轨迹上的任意两个不同时间步 |
| $[\epsilon, T]$ | 时间步的有效范围（$\epsilon$ 为小正数，避免奇点） |

训练方式：从预训练扩散模型蒸馏，或从头训练。支持 1-4 步高质量生成。

**与机器人实时控制的关键联系**：Consistency Distillation 将推理压缩到 1-2 步，使扩散策略达到 < 10ms 推理时延，令实时 RL 闭环控制成为可能。

### 5.4 Flow Matching：更简洁的数学

Flow Matching 用最优传输视角重新建立扩散过程：训练一个向量场，将噪声分布**沿直线**传输到数据分布。

- 扩散模型的概率流 ODE 是 Flow Matching 的特例（路径弯曲）
- Flow Matching 提供更直的路径，减少采样步数
- Stable Diffusion 3 已采用此框架

## 6. 条件控制：引导机制的统一框架

### 6.1 条件控制的本质：向分数函数叠加外部梯度

> **本节揭示了扩散模型最重要的"跨领域可迁移能力"**：无论是文本引导图像生成，还是奖励引导轨迹规划，都是同一种机制——**向分数函数叠加外部梯度信号**。

**Classifier Guidance**（Dhariwal 2021）：

$$\tilde{\varepsilon} = \varepsilon_\theta(x_t, t) - \sqrt{1-\bar\alpha_t} \cdot w \cdot \nabla_{x_t} \log p_\phi(c \mid x_t) \tag{17}$$

| 符号 | 含义 |
|------|------|
| $\tilde{\varepsilon}$ | 经分类器引导修正后的噪声预测 |
| $\varepsilon_\theta(x_t, t)$ | 无条件扩散模型的噪声预测 |
| $w$ | 引导强度（guidance scale），越大条件遵从度越强 |
| $\nabla_{x_t} \log p_\phi(c \mid x_t)$ | 独立分类器对条件 $c$ 的对数概率关于 $x_t$ 的梯度 |
| $\sqrt{1-\bar\alpha_t}$ | 尺度因子，使梯度信号与分数函数量纲对齐 |

**Classifier-Free Guidance**（Ho 2022）：

$$\hat{\varepsilon} = \varepsilon_\theta(x_t, t, \varnothing) + w \cdot \left(\varepsilon_\theta(x_t, t, c) - \varepsilon_\theta(x_t, t, \varnothing)\right) \tag{18}$$

| 符号 | 含义 |
|------|------|
| $\hat{\varepsilon}$ | 最终使用的混合引导噪声预测 |
| $\varepsilon_\theta(x_t, t, \varnothing)$ | 无条件（空条件）预测噪声 |
| $\varepsilon_\theta(x_t, t, c)$ | 有条件（条件为 $c$）预测噪声 |
| $w$ | 引导强度，$w=0$ 退化为无条件生成，$w$ 越大越贴近条件 |
| $c$ | 条件信息（如文本 embedding） |
| $\varnothing$ | 空条件（null condition），训练时随机丢弃真实条件所得 |

**RL 奖励引导**（Diffuser，Janner 2022）：

$$\hat{\mu} = \mu_\theta(x_t, t) + \alpha \cdot \Sigma \cdot \nabla_{x_t} J(x_t) \tag{19}$$

| 符号 | 含义 |
|------|------|
| $\hat{\mu}$ | 引导后修正的逆向均值 |
| $\mu_\theta(x_t, t)$ | 无引导的网络预测均值 |
| $\alpha$ | 奖励引导步长（类比 CFG 的 guidance scale $w$） |
| $\Sigma$ | 逆向过程的协方差矩阵（用于将梯度信号正确缩放到均值空间） |
| $\nabla_{x_t} J(x_t)$ | 奖励函数 $J$ 关于当前轨迹状态 $x_t$ 的梯度 |

**三种方式的统一本质**：

```
去噪方向 = 无条件去噪方向 + w × 外部信号梯度

          ↑                      ↑
   纯分布建模             条件引导（文本 / 分类器 / 奖励函数）
```

这个统一视角说明：**生成式扩散模型与决策式扩散模型之间没有本质鸿沟，只是外部信号的性质不同**（语义对齐 vs. 行为回报）。

### 6.2 CFG 的训练策略

训练时以概率 $p_{\text{uncond}}$（通常 10%-20%）随机将条件 $c$ 替换为空条件 $\varnothing$：

$$L_{\text{cfg}} = \mathbb{E}\left[\left\|\varepsilon - \varepsilon_\theta(x_t, t, c)\right\|^2\right] \qquad (c \text{ 有 } p_{\text{uncond}} \text{ 的概率被替换为 } \varnothing) \tag{20}$$

| 符号 | 含义 |
|------|------|
| $L_{\text{cfg}}$ | Classifier-Free Guidance 训练损失 |
| $\varepsilon$ | 真实噪声 |
| $\varepsilon_\theta(x_t, t, c)$ | 以条件 $c$ 为输入的噪声预测网络 |
| $p_{\text{uncond}}$ | 训练时将条件替换为空条件的概率（通常 10%–20%） |
| $\varnothing$ | 空条件（null token），替代真实条件 $c$ |


推理时 guidance scale $w$ 越大 → 条件遵从度越强，但多样性降低（trade-off）。

### 6.3 大规模条件生成：Latent Diffusion（Stable Diffusion）

```
输入图像 (512×512×3)
    │
    ▼
[预训练 VAE Encoder] ──→ 潜表示 z (64×64×4)   ← 压缩 64×
                              │
                              ▼
                    [扩散过程在潜空间进行]
                    [U-Net + Cross-Attention]
                    [条件: CLIP text embedding]
                              │
                              ▼
                    [预训练 VAE Decoder] ──→ 生成图像 (512×512×3)
```

核心优势：计算量降低 4-16×；在潜空间扩散，去除感知冗余；可灵活接入多种条件。

## 7. 从生成到决策：扩散模型的策略化

### 7.1 "生成动作"与"生成图像"的同构性

这是本文最核心的跨域洞见：**机器人动作生成问题与图像生成问题在数学上完全同构**。

| 维度 | 图像生成 | 机器人动作生成 |
|------|---------|-------------|
| **生成对象** | 像素矩阵 $x \in \mathbb{R}^{H \times W \times C}$ | 动作序列 $a \in \mathbb{R}^{T \times D_{action}}$ |
| **条件输入** | 文本 embedding | 视觉观测 + 机器人状态 |
| **多模态性** | 同一提示可生成多张不同但合理的图像 | 同一场景可生成多种不同但合理的抓取轨迹 |
| **骨干网络** | 2D U-Net / DiT | 1D U-Net / Transformer |
| **训练目标** | $\lVert\varepsilon - \varepsilon_\theta(x_t, t, \text{text})\rVert^2$ | $\lVert\varepsilon - \varepsilon_\theta(a_t, o, t)\rVert^2$ |

同构性的直接应用（Diffuser）：**将轨迹 $(s_0, a_0, s_1, a_1, ...)$ 拼接为 2D 数组，当作"图像"输入 U-Net**，复用图像生成的所有工程经验。

### 7.2 多模态能力：策略表示的核心价值

> 此处"多模态"指动作概率分布中有多个峰（mode）——同一状态下存在多种都合理但彼此不同的动作方案。

**具体例子**：桌上的杯子可以从左边抓也可以从右边抓——

```
RL（高斯策略）：  μ = (左抓 + 右抓) / 2 = 从中间去抓 → 失败
扩散模型策略：  有时采样到左抓，有时采样到右抓 → 每次都合理
```

**表达能力排序**：扩散 > GMM（高斯混合） > 高斯

### 7.3 Diffusion Policy：机器人模仿学习的范式转变

**Chi et al., RSS 2023**

将机器人策略 $\pi(a|o)$ 建模为条件扩散模型：

```
观测 o_t（图像 + 机器人状态）
    │
    ▼
[观测编码器]（ResNet / ViT）
    │
    ▼
观测特征 ──→ [条件扩散模型] ←── 噪声动作序列 a_T
                    │
                    ▼ (迭代去噪 K 步)
            去噪后的动作序列 a_0 = [a_t, ..., a_{t+H}]
                    │
                    ▼
        执行前几步动作，然后重新规划（receding horizon）
```

**为何扩散模型适合机器人策略（三大理由）**：

1. **多模态动作分布建模**：天然表达同一场景下多种合理动作
2. **时序一致性**：一次生成整条轨迹（而非逐步预测），避免抖动
3. **训练稳定性**：相比 GAN-based 策略（IBC），纯 MSE 损失更稳定

### 7.4 Diffuser：将规划转化为条件生成

**Janner et al., ICML 2022**

将**整条轨迹 $(s_0, a_0, s_1, a_1, ..., s_T, a_T)$** 作为生成目标，通过 inpainting 实现条件约束：

```
生成目标（2D 数组视为"图像"）：
┌─────────────────────────────────────────────┐
│  s₀  a₀  s₁  a₁  s₂  a₂  ...  sT  aT    │
└─────────────────────────────────────────────┘
     ▲                              ▲
     │                              │
   固定为当前状态              可固定为目标状态
   (inpainting)               (inpainting)
```

奖励引导（与 CFG 同构的机制）：

$$\hat{\mu} = \mu_\theta(x_t, t) + \alpha \cdot \Sigma \cdot \nabla_{x_t} J(x_t) \tag{21}$$

| 符号 | 含义 |
|------|------|
| $\hat{\mu}$ | 奖励引导后的逆向均值（同式 $(19)$，此处用于轨迹规划） |
| $\mu_\theta(x_t, t)$ | 无引导时的扩散模型预测均值 |
| $\alpha$ | 奖励引导步长 |
| $\Sigma$ | 逆向过程协方差矩阵 |
| $J(x_t)$ | 对当前轨迹片段 $x_t$ 的累积回报估计函数 |
| $\nabla_{x_t} J(x_t)$ | 回报函数关于 $x_t$ 的梯度，指向高回报方向 |

其中 $J(x_t)$ 是对轨迹回报的估计。这本质上是**将 RL 的优化目标嵌入了扩散模型的采样过程**。

## 8. 扩散与强化学习：互补与融合

### 8.1 互补性的根本来源

| 维度 | 强化学习（RL） | 扩散模型 |
|------|-------------|---------|
| **核心目标** | 最大化累积奖励 | 建模数据分布并生成样本 |
| **策略表示** | 参数化 $\pi_\theta(a \mid s)$（通常高斯） | 通过去噪过程隐式定义 |
| **数据来源** | 在线交互 or 离线数据集 | 离线演示数据 |
| **多模态能力** | 弱（高斯策略难以表达多峰） | **强（天然建模多模态分布）** |
| **优化信号** | 奖励函数 $r(s, a)$ | 去噪损失 $\lVert\epsilon - \epsilon_\theta\rVert^2$ |
| **探索能力** | **强（主动探索未知区域）** | 弱（依赖训练数据覆盖） |
| **样本效率** | 低（需大量交互） | **高（离线数据即可）** |
| **安全性** | 探索阶段可能危险 | **不主动探索，更安全** |

**一句话总结**：扩散模型擅长"生成多样且合理的行为"，RL 擅长"通过奖励信号优化行为"。两者结合 = 既有好的行为先验，又能持续优化。

### 8.2 四种融合范式

#### 范式一：扩散模型作为 RL 的策略表示

用扩散模型替换传统高斯策略，使策略天然多模态：

$$\text{传统 RL:} \;\; \pi(a|s) = \mathcal{N}(\mu_\theta(s), \sigma^2) \qquad \text{Diffusion-QL:} \;\; a_K \sim \mathcal{N}(0,I) \xrightarrow{\text{去噪 K 步}} a_0 \tag{22}$$

| 符号 | 含义 |
|------|------|
| $\pi(a \mid s)$ | 传统 RL 的高斯策略分布 |
| $\mu_\theta(s)$ | 策略网络输出的动作均值 |
| $\sigma^2$ | 策略方差（通常固定或学习） |
| $a_K \sim \mathcal{N}(0, I)$ | Diffusion-QL 从纯高斯噪声初始化的动作 |
| $a_0$ | 经过 $K$ 步去噪后的最终动作（隐式定义了策略分布） |

训练目标：

$$\mathcal{L} = \underbrace{\mathbb{E}\left[\|\epsilon - \epsilon_\theta(a_t, s, t)\|^2\right]}_{\text{行为克隆项（扩散损失）}} - \alpha \underbrace{\mathbb{E}[Q_\phi(s, a_0)]}_{\text{RL 项（Q 值引导）}} \tag{23}$$

| 符号 | 含义 |
|------|------|
| $\mathcal{L}$ | Diffusion-QL 总训练损失 |
| $\epsilon$ | 真实噪声 |
| $\epsilon_\theta(a_t, s, t)$ | 以状态 $s$ 为条件的动作噪声预测网络 |
| $a_t$ | $t$ 时刻的含噪动作 |
| $s$ | 当前环境状态 |
| $Q_\phi(s, a_0)$ | 参数为 $\phi$ 的 Q 函数（动作价值），对动作 $a_0$ 评分 |
| $\alpha$ | 平衡行为克隆项与 Q 值引导项的权重系数 |

| 代表工作 | 特点 |
|---------|------|
| Diffusion-QL（Wang 2023） | 离线 RL 策略约束，扩散损失 + Q 值正则 |
| IDQL（Hansen-Estruch 2023） | 解耦扩散策略和 Q 函数训练，重要性采样加权 |
| QGPO（Lu 2023） | Q 函数做 classifier guidance |

#### 范式二：扩散模型作为规划器 + RL 奖励引导

```
传统 Model-Based RL:
  世界模型 p(s'|s,a) → 前向模拟 → CEM / MPPI 规划

Diffuser 方式:
  扩散模型直接生成完整轨迹 τ = (s₀,a₀,...,sT,aT)
  通过奖励梯度引导轨迹向高回报方向
```

**关键优势**：把世界模型和规划器统一成一个生成过程，无需分开训练。

| 代表工作 | 引导机制 |
|---------|---------|
| Diffuser（Janner 2022） | Classifier guidance（奖励梯度） |
| Decision Diffuser（Ajay 2023） | Classifier-free guidance（回报条件） |

#### 范式三：RL 微调扩散策略（DPPO）

**两阶段流程**：

```
阶段1: 演示数据 → 训练 Diffusion Policy（行为克隆预训练）
阶段2: RL 微调（PPO + 扩散策略）
       关键问题：多步去噪如何计算策略梯度？
       解决方案：将去噪链视为"虚拟 MDP"，PPO 梯度沿去噪链反向传播
```

**虚拟 MDP 视角（DPPO 核心创新）**：

```
真实环境状态 s → a_K ~ N(0,I)
                    │
                    ▼  去噪步骤 K（虚拟 MDP 第一步）
                 a_{K-1}
                    │  ...（每步视为一个虚拟状态转移）
                    ▼  去噪步骤 1（虚拟 MDP 最后一步）
                   a_0 → 在真实环境中执行 → 获得奖励 r
                         ↑___________________________↑
                    PPO 梯度沿整个去噪链反向传播
```

**优势**：模仿学习提供好的初始化（避免 RL 从零探索），RL 微调使策略超越演示质量，同时保留多模态能力。

#### 范式四：扩散模型作为世界模型服务于 RL

```
真实少量数据 → 训练扩散世界模型 p_θ(s', r | s, a)
                      ↓
在扩散世界模型中"想象"大量轨迹
                      ↓
用想象轨迹做 RL 训练（类似 Dreamer / MBPO）
                      ↓
策略部署到真实环境
```

**优势**：大幅减少真实交互次数；扩散世界模型生成的轨迹更逼真；可生成多样化场景做 domain randomization。

代表工作：UniSim（视频扩散做世界模拟器）、SuSIE（扩散模型生成子目标图像）

### 8.3 融合范式全谱系

```
          RL 与 Diffusion 融合谱系

纯模仿学习 ◄─────────────────────────────► 纯强化学习
(Diffusion Policy)                       (SAC/PPO)

    │          │            │           │
    ▼          ▼            ▼           ▼
 扩散策略   奖励引导      RL 微调    Diffusion 为
 (BC only)  推理时注入    扩散策略   RL 提供世界模型
            奖励梯度      (DPPO)    (Model-based RL)
            (Diffuser)

◄── 更依赖数据质量          更依赖奖励设计 ──►
◄── 更安全/保守             更激进/可能更优 ──►
◄── 实现更简单              实现更复杂     ──►
```

## 9. 机器人与自动驾驶：应用全景

### 9.1 机器人：层次化协作框架

```
┌────────────────────────────────────────────────────────────┐
│  高层：任务规划层（LLM / Task Planner）                      │
│  输入: 自然语言指令"把杯子放到盘子上"                         │
│  输出: 子目标序列 [抓取杯子]→[移动到盘子上方]→[放下杯子]      │
└────────────────────────────┬───────────────────────────────┘
                             ▼
┌────────────────────────────────────────────────────────────┐
│  中层：运动规划层（Diffusion Model）                         │
│  输入: 子目标 + 视觉观测（RGB/点云）+ 机器人状态              │
│  输出: 动作轨迹 [a_t, a_{t+1}, ..., a_{t+H}]               │
│  优势: 多模态动作生成、时序一致性、平滑轨迹                    │
└────────────────────────────┬───────────────────────────────┘
                             ▼
┌────────────────────────────────────────────────────────────┐
│  底层：控制执行层（RL / 传统控制）                             │
│  输入: 目标轨迹 + 实时传感器反馈（力/力矩、关节状态）           │
│  输出: 关节力矩 / 电机指令                                   │
│  作用: 鲁棒执行、力控制、sim-to-real 适应                     │
└────────────────────────────────────────────────────────────┘
```

| 层级 | 核心问题 | 时间尺度 | 扩散模型角色 | RL 角色 |
|------|---------|---------|------------|--------|
| 任务层 | 做什么 (What) | 秒~分钟 | - | LLM/规划器 |
| 规划层 | 怎么做 (How) | 100ms~秒 | **主导**：生成动作轨迹 | 辅助：Q 值引导 |
| 控制层 | 执行 (Execute) | 1~10ms | 辅助：提供参考轨迹 | **主导**：实时控制与适应 |

### 9.2 机器人应用汇总

| 应用方向 | 代表工作 | 扩散模型角色 | 条件输入 |
|---------|---------|------------|---------|
| 模仿学习/策略 | Diffusion Policy | 动作生成器 | 视觉 + 本体感知 |
| 轨迹规划 | Diffuser | 全轨迹规划器 | 起始/目标状态 + 奖励 |
| 抓取规划 | SE(3)-DiffusionFields | 抓取位姿生成 | 点云 |
| 跨具身策略 | UMI | 通用操控策略 | 视觉 + 相对位姿 |
| 数据增强 | GenAug, RoboGen | 生成训练数据 | 场景描述 |
| RL 策略表示 | Diffusion-QL | 策略分布 | 状态 + Q 值引导 |
| RL 微调 | DPPO | 可 RL 优化的策略 | 状态 + 在线奖励 |
| 世界模型 | UniSim, SuSIE | 环境动力学模拟 | 动作 + 历史观测 |
| 人形机器人 | HumanPlus | 全身动作生成 | 人类动作捕捉 |

### 9.3 自动驾驶应用汇总

| 应用方向 | 代表工作 | 输入 | 输出 |
|---------|---------|------|------|
| 场景图像生成 | MagicDrive | 3D Box + BEV + 文本 | 多视角驾驶图像 |
| 驾驶视频生成 | DriveDreamer | HDMap + 文本 | 时序一致驾驶视频 |
| 运动规划 | DiffusionDrive | 传感器 + 地图 | 自车规划轨迹（2 步去噪） |
| 多智能体预测 | MotionDiffuser | 历史轨迹 + 地图 | 联合未来轨迹 |
| 端到端驾驶 | GenAD | 传感器原始数据 | 感知+预测+规划统一 |
| 世界模型 | GAIA-2 | 历史帧+动作 | 未来驾驶视频 |
| 仿真 | UniSim | 场景编辑指令 | 多传感器仿真 |

**自动驾驶 vs. 机器人的关键共性**：两个领域都在用扩散模型解决"多模态行为生成 + 安全约束"问题，区别只在于动作空间的定义（车道轨迹 vs. 关节动作）。

## 10. 方法选择指南与前沿趋势

### 10.1 场景 → 方法决策树

```
开始
 │
 ├─ 有高质量演示数据？
 │   ├─ 是 → 演示是否足够覆盖所有场景？
 │   │        ├─ 是 → Diffusion Policy（纯模仿学习）
 │   │        └─ 否 → Diffusion Policy + RL 微调（DPPO）
 │   └─ 否 → 有仿真器？
 │            ├─ 是 → 扩散世界模型 + RL 训练
 │            └─ 否 → 传统 RL（在线学习）
 │
 ├─ 动作空间是否多模态？
 │   ├─ 是 → 必须用扩散模型（或 GMM 等多模态策略）
 │   └─ 否 → 高斯策略的传统 RL 可能足够
 │
 └─ 实时性要求？
     ├─ < 1ms  → 传统控制 / 轻量 RL
     ├─ 1-10ms → Consistency Distillation 后的扩散策略
     ├─ 10-100ms → 标准 Diffusion Policy
     └─ > 100ms → 全步数扩散（质量最高）
```

### 10.2 综合选择矩阵

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 大量高质量演示 | Diffusion Policy | 直接拟合演示分布 |
| 演示少但有仿真器 | RL + 扩散世界模型 | 在想象中大量训练 |
| 需要超越演示水平 | Diffusion Policy + RL 微调 | 扩散初始化 + RL 优化 |
| 动作分布简单（单峰） | 传统 RL | 无需多模态建模 |
| 实时性要求极高 | Consistency Policy | 1-2 步推理 |
| 多智能体协调 | 扩散规划器 + 分布式 RL | 扩散建模联合分布 |
| 离线数据质量参差不齐 | 扩散正则化的离线 RL | 约束策略在数据支撑内 |
| 长时序复杂任务 | LLM + Diffusion + RL 层次化 | 各层各司其职 |

### 10.3 前沿趋势（2024-2025）

**1. 直接对扩散策略做策略梯度优化**

- DPPO（Allen 2024）：PPO + Diffusion Policy，去噪链视为虚拟 MDP
- RFGP：奖励引导的 Flow Matching 策略
- 核心：将扩散模型的去噪过程展开成可微分计算图，RL 梯度端到端传播

**2. Foundation Model + 扩散 + RL（$\pi_0$ 架构）**

```
互联网规模预训练（VLM）
        │
        ▼
视觉-语言-动作基础模型
        │
        ▼
扩散动作头（Diffusion Action Head）← 生成多模态动作
        │
        ▼
RL Post-Training ← 针对具体任务/机器人微调
        │
        ▼
通用机器人策略
```

**3. 实时扩散控制**

通过 Consistency Distillation 把去噪压缩到 1-2 步，实现 < 10ms 推理。代表：Consistency Policy（Prasad 2024）。

**4. 扩散世界模型 + RL 大规模训练**

用视频扩散模型做世界模型（UniSim、Genie 2），RL agent 在"想象"的视频世界中大规模学习。类比 AlphaGo 在模拟器中自我博弈，但世界模型是学习到的。

**5. 安全约束下的扩散 + RL**

扩散模型生成候选轨迹 → RL 价值函数 + 安全 critic 筛选 → 形式化安全保障（CBF / 可达性分析）约束扩散采样。

### 10.4 当前主要挑战

| 挑战 | 现状 | 研究方向 |
|------|------|---------|
| **推理速度** | 仍需多步（2-50 步） | Consistency Models、蒸馏、Flow Matching |
| **实时性** | 机器人/自驾需毫秒级响应 | 截断扩散、缓存机制、硬件加速 |
| **长时序一致性** | 长轨迹/长视频一致性难保证 | 层次化扩散、时空注意力 |
| **物理合理性** | 生成结果可能违反物理定律 | 物理约束引导、物理模拟器结合 |
| **安全保障** | 生成式策略安全性难以形式化验证 | 安全引导、约束优化、运行时监控 |

> **总结**：扩散模型的本质是通过学习分数函数来建模复杂数据分布，并通过梯度引导来控制生成方向。这一机制天然支持三个层次的应用：
（1）**静态分布建模**——生成图像/轨迹；
（2）**条件生成**——CFG / 奖励引导，注入外部信号；
（3）**在线优化**——RL 微调，让分布向高回报区域持续演进。
这三个层次依次递进，构成了从图像生成到自主决策的完整路径。
